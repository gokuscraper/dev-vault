💻 Mobile vs Desktop Behavior

Mobile Devices

  • Physical accelerometer drives orientation.type
  • orientation.lock() works in fullscreen
  • angle reflects physical rotation (0, 90, 180, 270)
  • portrait-secondary / landscape-secondary available
  • "change" event fires on physical rotation
  • Some tablets default to landscape-primary

Desktop Browsers

  • Orientation based on window dimensions
  • orientation.lock() usually throws NotSupportedError
  • angle is typically always 0
  • Only portrait-primary or landscape-primary reported
  • "change" event fires on window resize crossing threshold
  • DevTools device mode can simulate mobile behavior

📚 API Reference

// Reading current orientation const orientation = screen.orientation; console.log(orientation.type); // "portrait-primary" console.log(orientation.angle); // 0 // Listening for changes screen.orientation.addEventListener('change', (e) => { console.log(`New: ${e.target.type} at ${e.target.angle}deg`); }); // Locking orientation (requires fullscreen) await document.documentElement.requestFullscreen(); await screen.orientation.lock('landscape'); // Unlocking screen.orientation.unlock(); // Lock type options: // "any" - any orientation // "natural" - device's natural orientation // "landscape" - landscape-primary or landscape-secondary // "portrait" - portrait-primary or portrait-secondary // "portrait-primary" - upright portrait // "portrait-secondary" - upside-down portrait // "landscape-primary" - default landscape // "landscape-secondary" - inverted landscape
