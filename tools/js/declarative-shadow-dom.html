Template shadowrootmode
With shadowrootmode="open", the shadow root is accessible via element.shadowRoot. This is the most common mode for declarative shadow DOM.
HTML (Server-rendered)
<my-card> <template shadowrootmode="open"> <style> :host { display: block; border: 1px solid #ccc; padding: 1rem; border-radius: 8px; } h2 { color: #4fc3f7; } </style> <h2><slot name="title">Default Title</slot></h2> <slot>Default content</slot> </template> <span slot="title">My Card</span> <p>This content is projected into the default slot.</p> </my-card>
Live Preview
JavaScript access
const card = document.querySelector('my-card'); card.shadowRoot; // ShadowRoot (accessible!) card.shadowRoot.querySelector('h2'); // <h2>...</h2>
With shadowrootmode="closed", element.shadowRoot returns null. The shadow root cannot be accessed from outside. Useful for encapsulating implementation details.
HTML
<secret-box> <template shadowrootmode="closed"> <style> :host { display: block; padding: 1rem; background: #1e1e1e; border-radius: 8px; } .secret { color: #ef5350; } </style> <p class="secret">Internal implementation hidden</p> <slot></slot> </template> <p>Visible slotted content</p> </secret-box>
Live Preview
JavaScript access
const box = document.querySelector('secret-box'); box.shadowRoot; // null (not accessible!) // Cannot reach internal DOM from outside

SSR Hydration Flow
Declarative Shadow DOM enables server-side rendering of shadow DOM components. The browser parses the template and creates the shadow root during HTML parsing, before any JavaScript runs.

<!-- Server sends this HTML --> <my-counter> <template shadowrootmode="open"> <style>button { padding: 0.5rem 1rem; }</style> <button>Count: <span id="val">0</span></button> </template> </my-counter> <!-- Browser creates shadow root during parsing --> <!-- Content is visible BEFORE JavaScript loads --> <script> // JS hydrates: adds event listeners, state management class MyCounter extends HTMLElement { connectedCallback() { // Shadow root already exists from DSD! const btn = this.shadowRoot.querySelector('button'); let count = 0; btn.onclick = () => { this.shadowRoot.getElementById('val').textContent = ++count; }; } } customElements.define('my-counter', MyCounter); </script>
Slot Projection
Named slots let you project specific content into designated areas of a shadow DOM template. The default slot catches any content without a slot attribute.
Component Template
<article-card> <template shadowrootmode="open"> <style> :host { display: block; border: 1px solid #333; border-radius: 8px; overflow: hidden; } header { background: #1e1e1e; padding: 0.75rem; } .body { padding: 1rem; } footer { background: #1e1e1e; padding: 0.5rem; font-size: 0.8rem; color: #888; } </style> <header> <slot name="title">Untitled</slot> </header> <div class="body"> <slot>No content provided</slot> </div> <footer> <slot name="footer">No footer</slot> </footer> </template> <!-- Projected content --> <h3 slot="title">Article Title</h3> <p>This goes into the default slot.</p> <span slot="footer">Published today</span> </article-card>

Style Encapsulation Comparison
Without Shadow DOM
<!-- Global styles leak in --> <style> .card-title { color: red; font-size: 24px; } </style> <div class="card"> <h3 class="card-title">I'm red</h3> </div> <!-- Any .card-title on the page is affected -->
With Shadow DOM
<!-- Styles are encapsulated --> <my-card> <template shadowrootmode="open"> <style> .card-title { color: #4fc3f7; font-size: 18px; } </style> <h3 class="card-title">I'm blue</h3> </template> </my-card> <!-- .card-title only applies inside this shadow root -->
adoptedStyleSheets
adoptedStyleSheets lets you share CSSStyleSheet objects across multiple shadow roots without duplicating CSS text. More efficient than inline styles.
// Create a shared stylesheet const sheet = new CSSStyleSheet(); sheet.replaceSync(` :host { display: block; padding: 1rem; } button { background: #4fc3f7; color: #000; border: none; padding: 0.5rem 1rem; border-radius: 8px; cursor: pointer; } `); // Apply to multiple shadow roots class MyButton extends HTMLElement { constructor() { super(); const shadow = this.attachShadow({ mode: 'open' }); shadow.adoptedStyleSheets = [sheet]; // Share, not copy! shadow.innerHTML = '<button><slot></slot></button>'; } } // With Declarative Shadow DOM: // After parsing, hydrate with adopted stylesheets class HydratedComponent extends HTMLElement { connectedCallback() { // Shadow root exists from DSD template this.shadowRoot.adoptedStyleSheets = [sheet]; } } // Benefits: // - Single stylesheet instance shared across all components // - Reduced memory usage // - Dynamic updates propagate to all instances sheet.replaceSync('button { background: #66bb6a; }'); // All components update instantly!

Serialization (getHTML)
getHTML({ serializableShadowRoots: true }) serializes the DOM including shadow roots back to HTML with <template shadowrootmode> tags.
// Get HTML including shadow DOM const html = element.getHTML({ serializableShadowRoots: true }); // Or specify which shadow roots to include const html2 = element.getHTML({ shadowRoots: [element.shadowRoot] }); // Result includes DSD templates: // <my-card> // <template shadowrootmode="open" // shadowrootserializable> // <style>...</style> // <slot></slot> // </template> // Content here // </my-card>
