Text Fragment URL Builder

Create a URL that links directly to specific text on a page using the Text Fragments API (#:~:text=).

URL Anatomy

https://example.com/article#:~:text=important%20section

Breakdown:
  https://example.com/article  ← Base URL
  #                            ← Fragment identifier start
  :~:                          ← Fragment directive delimiter
  text=                        ← Text fragment directive
  important%20section          ← URL-encoded target text

Fragment Directive Syntax

PatternDescriptionExample

Prefix & Suffix Context

Prefix and suffix context help disambiguate when the same text appears multiple times on a page. The context text is not highlighted — it only helps the browser find the right occurrence.

The quick brown fox jumps over the lazy dog. The quick brown fox then runs into the forest. Later, the quick brown fox returns home.

Scenario: Target the second "quick brown fox"

#:~:text=quick%20brown%20fox
→ Highlights the FIRST occurrence
#:~:text=dog.%20The-,quick%20brown%20fox
→ Highlights the SECOND occurrence

The Evolution of Web APIs

The web platform has evolved dramatically over the past decade. From simple document viewing to complex application development, browsers now provide hundreds of APIs that enable rich, interactive experiences.

One of the most impactful changes has been the introduction of service workers, which enable offline functionality and background processing. Combined with the Cache API, developers can create applications that work reliably even without network connectivity.

The Fetch API replaced the aging XMLHttpRequest, providing a cleaner promise-based interface for network requests. It supports streaming responses, request and response objects, and integrates seamlessly with service workers.

Web Components brought native component architecture to the browser with Custom Elements, Shadow DOM, and HTML Templates. This reduced the need for framework-specific component models and improved interoperability.

Modern CSS has also seen revolutionary changes. CSS Grid and Flexbox transformed layout capabilities. Custom Properties (CSS Variables) enabled dynamic theming. Container Queries finally allowed component-level responsive design.

The Text Fragments API is one of the newer additions, allowing precise deep linking to specific text content on any page, even without anchor IDs. This improves citation, sharing, and accessibility.

Looking ahead, APIs like the View Transitions API, Navigation API, and Speculation Rules are set to further blur the line between web applications and native experiences.

::target-text Styling

The ::target-text pseudo-element allows you to customize the appearance of text fragment highlights. Browser support is limited — Chrome 89+ supports it.

Generated CSS

  color: #000000;
  background-color: #ffd700;
  text-decoration: none;
}

Security Constraints

  • Only works on top-level navigations (not iframes)
  • Requires user activation (click, tap) to navigate
  • Cross-origin text fragments are restricted for privacy
  • Fragment directive (:~:) is stripped from location.hash
  • Not accessible via JavaScript for privacy reasons
