🎯 Eagerness Levels

Choose how aggressively the browser should speculate. Click to select.
🔥

immediate

Speculate as soon as rules are observed. No user signal needed.

⚡

eager

Speculate on any slight signal from the user. Default for list rules.

⏳

moderate

Speculate on hover for 200ms. Balanced approach.

🛡️

conservative

Only on mouse/touch down. Saves resources but less benefit.

Tip: moderate is recommended for document rules. eager is the default for list rules.

📖 Document vs List Rules

List Rules

  • ✅ Explicit URL list
  • ✅ Works with any URL
  • ✅ Default: eager
  • ✅ Best for known pages
  • ❌ Manual maintenance

Document Rules

  • ✅ CSS selector-based
  • ✅ URL pattern matching
  • ✅ Default: conservative
  • ✅ Dynamic, auto-discovers
  • ❌ May speculate too much
Prerender vs Prefetch: Prerender fully renders the page in a hidden tab (instant navigation). Prefetch only downloads resources. Prerender uses more bandwidth/CPU but gives the best UX.

🛠️ Chrome DevTools: Speculations Tab

Chrome DevTools has a dedicated Speculations tab under the Application panel for debugging speculation rules.
  1. Open Chrome DevTools (F12 or Cmd+Option+I)
  2. Go to the Application panel
  3. Find Speculative loads in the left sidebar under "Background services"
  4. Click Speculation Rules to see all active rules
  5. Click Speculations to see the status of each speculated URL
  6. Status values: Ready (prerendered), Pending, Failure
  7. Click a failure to see the reason (e.g., same-site restriction, limit reached)
Limits: Chrome allows up to 10 prefetch and 2 prerender speculations at a time (eager). Conservative/moderate allow more (10 prerender).
