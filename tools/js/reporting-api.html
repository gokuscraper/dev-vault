Report Types

Overview of all report types supported by the Reporting API ecosystem.

Deprecation

ReportingObserver

Generated when a deprecated API feature is used. Includes the feature name, message, and source location.

Trigger: Use deprecated APIs like document.all in strict contexts
{ type: "deprecation", body: { id: "feature-name", message: "...", sourceFile: "...", lineNumber: 42, columnNumber: 10 } }

Intervention

ReportingObserver

Generated when the browser overrides a behavior for performance, security, or UX reasons.

Trigger: Heavy ads, autoplay blocked, slow scripts
{ type: "intervention", body: { id: "intervention-name", message: "...", sourceFile: "...", lineNumber: 0, columnNumber: 0 } }

CSP Violation

Report-To Header

Content Security Policy violation reports. Sent to endpoints configured via report-to directive in CSP header.

Trigger: CSP violations (inline scripts, unauthorized resources)
{ type: "csp-violation", body: { documentURL: "...", blockedURL: "...", violatedDirective: "script-src", effectiveDirective: "...", disposition: "enforce" } }

Crash

Report-To Header

Generated when a page crashes (e.g., out of memory). Sent when the page is later revisited.

Trigger: Page crash, out-of-memory
{ type: "crash", body: { reason: "oom" | "unresponsive" } }

Permissions Policy

Report-To Header

Violations of Permissions Policy (formerly Feature Policy). Reports blocked API access.

Trigger: Using blocked features (camera, geolocation)
{ type: "permissions-policy-violation", body: { featureId: "camera", disposition: "enforce", sourceFile: "..." } }

COEP/COOP

Report-To Header

Cross-Origin Embedder/Opener Policy violations. Important for SharedArrayBuffer access.

Trigger: Cross-origin resource without CORS headers
{ type: "coep", body: { type: "corp", blockedURL: "...", disposition: "enforce" } }

Network Error Logging (NEL)

NEL uses the Reporting API infrastructure to report network-level errors. Configured via HTTP headers.

How NEL Works

NEL captures network errors that happen below the application layer: DNS failures, TCP timeouts, TLS errors, and HTTP errors. It requires the Report-To header for endpoint configuration.

// Server response headers: // 1. Configure reporting endpoint Report-To: { "group": "network-errors", "max_age": 86400, "endpoints": [{"url": "https://example.com/nel"}] } // 2. Enable NEL NEL: { "report_to": "network-errors", "max_age": 86400, "include_subdomains": true, "success_fraction": 0.0, "failure_fraction": 1.0 }
