✎ Basic EditContext

Attach an EditContext to a DOM element to receive text input without contenteditable. The element becomes an input surface while you control rendering.

Minimal Example

Click the editor below and type. The EditContext handles text input; we render it manually.

Click here and start typing...
// 1. Create an EditContext
const editContext = new EditContext();

// 2. Attach it to a DOM element
const editor = document.getElementById('editor');
editor.editContext = editContext;

// 3. Listen for text updates
editContext.addEventListener('textupdate', (e) => {
  // e.text — the new text
  // e.updateRangeStart — start of replaced range
  // e.updateRangeEnd — end of replaced range
  // e.selectionStart / e.selectionEnd — new cursor
  renderText();
});

function renderText() {
  editor.textContent = editContext.text;
}

▣ Custom Canvas Editor

EditContext enables text input on a <canvas> element — impossible with contenteditable. You render glyphs, cursor, and selection manually.

Canvas Text Editor

Click the canvas and type. Text is rendered via Canvas 2D API. Arrow keys and backspace work.

Ready 
const canvas = document.querySelector('canvas');
const ctx = canvas.getContext('2d');
const ec = new EditContext();
canvas.editContext = ec;

ec.addEventListener('textupdate', () => draw());

function draw() {
  ctx.clearRect(0, 0, canvas.width, canvas.height);
  ctx.font = '16px monospace';
  ctx.fillStyle = '#e2e8f0';

  // Draw text
  ctx.fillText(ec.text, 16, 32);

  // Draw cursor
  const before = ec.text.slice(0, ec.selectionStart);
  const x = 16 + ctx.measureText(before).width;
  ctx.fillStyle = '#6366f1';
  ctx.fillRect(x, 16, 2, 20);
}

B Text Formatting

EditContext does not provide built-in formatting — you control rendering entirely. Apply bold, italic, or any style to selection ranges with custom rendering logic.

Rich Text Editor

Type text, select a range with Shift+Arrow, then click a formatting button. Formatting is tracked as character-level metadata.

Type here and format your text...
// Store format metadata per character
const charFormats = []; // [{bold, italic, underline, highlight}]

function applyFormat(type) {
  const { selectionStart, selectionEnd } = editContext;
  for (let i = selectionStart; i < selectionEnd; i++) {
    charFormats[i] = charFormats[i] || {};
    charFormats[i][type] = !charFormats[i][type];
  }
  renderFormatted();
}

function renderFormatted() {
  // Build spans with format classes
  editor.innerHTML = editContext.text
    .split('')
    .map((ch, i) => {
      const f = charFormats[i] || {};
      let style = '';
      if (f.bold) style += 'font-weight:700;';
      if (f.italic) style += 'font-style:italic;';
      if (f.underline) style += 'text-decoration:underline;';
      if (f.highlight) style += 'background:rgba(245,158,11,0.3);';
      return `<span style="${style}">${ch}</span>`;
    }).join('');
}

{} API Explorer

Interactive reference for every EditContext property, method, and event.

editContext.text : string (read-only)
The current text content managed by the EditContext. Updated via textupdate events or updateText().
editContext.selectionStart : number (read-only)
Start index of the current selection. Equals selectionEnd when no text is selected (cursor position).
editContext.selectionEnd : number (read-only)
End index of the current selection.
editContext.characterBoundsRangeStart : number (read-only)
Start of the range for which character bounds have been provided via updateCharacterBounds().
element.editContext : EditContext | null
Assign an EditContext to any focusable element to make it an editing host. Set to null to detach.
editContext.updateText(rangeStart, rangeEnd, newText)
Programmatically replace text in [rangeStart, rangeEnd) with newText. Use this for undo/redo or programmatic edits.
editContext.updateSelection(start, end)
Set the selection range. If start === end, positions the cursor. Call this after programmatic text changes.
editContext.updateControlBounds(domRect)
Tell the OS where the editing UI is, so IME candidate windows appear in the right position.
editContext.updateSelectionBounds(domRect)
Tell the OS where the current selection is, for IME positioning.
editContext.updateCharacterBounds(rangeStart, bounds[])
Provide bounding rects for characters in the composition range. Required for proper IME overlay positioning.
editContext.attachedElements()
Returns an array of elements this EditContext is attached to.
editContext.characterBounds()
Returns the array of DOMRect objects previously set via updateCharacterBounds().
textupdate
Fired when the user types, deletes, or pastes. Properties: text, updateRangeStart, updateRangeEnd, selectionStart, selectionEnd.
textformatupdate
Fired during IME composition. Call e.getTextFormats() to get underline/highlight info for composition text.
characterboundsupdate
Fired when the OS needs character bounds (for IME). Respond by calling updateCharacterBounds().
compositionstart
Fired when IME composition begins.
compositionend
Fired when IME composition is committed or cancelled.

When to Use EditContext

Good Fit

  • Code editors with syntax highlighting
  • Canvas/WebGL-based text rendering
  • Custom IME handling requirements
  • Very large document editing
  • Game UI text fields
  • Collaborative editing engines

Poor Fit

  • Simple forms and inputs
  • Basic rich text (use contenteditable)
  • When accessibility is critical (adds burden)
  • Cross-browser requirement (Firefox/Safari)
  • Rapid prototyping

Feature Detection

if ('EditContext' in window) {
  // EditContext API is available
  const ec = new EditContext();
  element.editContext = ec;
} else {
  // Fallback to contenteditable or textarea
  element.contentEditable = 'true';
}

Your Browser

Polyfill / Fallback Strategy

There is no polyfill for EditContext — it requires deep integration with the OS input pipeline. The recommended approach is progressive enhancement: use EditContext when available, fall back to a hidden <textarea> for other browsers. Libraries like CodeMirror 6 have used this textarea-proxy pattern successfully for years. EditContext simply provides a cleaner, more capable version of the same idea.
