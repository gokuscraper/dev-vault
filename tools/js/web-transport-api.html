Datagrams vs Streams
Datagram Unreliable Datagrams
Fire-and-forget messages. No delivery guarantee, no ordering. Best for real-time data where latest value matters most.
Client ↔ Server
// Send datagram const writer = transport.datagrams.writable.getWriter(); await writer.write(new Uint8Array([1, 2, 3])); // Receive datagrams const reader = transport.datagrams.readable.getReader(); const { value } = await reader.read();
Use for: game state, telemetry, live cursor position, audio/video frames
Unidirectional One-Way Streams
Reliable, ordered delivery in one direction. Good for sending files, logs, or events.
Sender → Receiver
// Client opens unidirectional stream const stream = await transport.createUnidirectionalStream(); const writer = stream.getWriter(); await writer.write(data); await writer.close(); // Server receives const reader = transport.incomingUnidirectionalStreams.getReader(); const { value: stream } = await reader.read();
Use for: file uploads, event streams, log shipping
Bidirectional Two-Way Streams
Reliable, ordered delivery in both directions. Like a TCP connection but multiplexed without head-of-line blocking.
Client ↔ Server
// Open bidirectional stream const stream = await transport.createBidirectionalStream(); // Write to server const writer = stream.writable.getWriter(); await writer.write(request); // Read response const reader = stream.readable.getReader(); const { value } = await reader.read();
Use for: request/response, RPC, interactive protocols
Multiplexing Advantage
Multiple independent streams share one connection. A slow stream does not block others (no head-of-line blocking at transport layer).

Why WebTransport?
0-RTT Connection
QUIC enables sending data with the first packet on reconnection. No 3-way handshake needed.
No Head-of-Line Blocking
Lost packets on one stream don't block other streams. Each stream is independent at the transport layer.
Unreliable Datagrams
Send fire-and-forget messages for real-time data where latest value matters more than every value.
Connection Migration
QUIC connections survive IP changes (WiFi to cellular). Uses connection IDs instead of IP:port tuples.
Built-in Encryption
TLS 1.3 is mandatory in QUIC. All data is encrypted. No unencrypted fallback possible.
Stream Multiplexing
Open thousands of independent streams over a single connection. Each with its own flow control.
Error Handling & Configuration
Connection Failed
Server unreachable, certificate mismatch, or HTTP/3 not supported
try { const transport = new WebTransport(url); await transport.ready; } catch (e) { console.error('Connection failed:', e); // Fallback to WebSocket }
Connection Closed
Server closed connection or network error
transport.closed.then(info => { console.log(`Closed: code=${info.closeCode}, reason=${info.reason}`); }).catch(err => { console.error('Connection closed with error:', err); });
Stream Error
Individual stream can fail without affecting others
try { const stream = await transport.createBidirectionalStream(); // ... use stream } catch (e) { // Stream-level error, connection still alive console.error('Stream error:', e); }
During development, you can use self-signed certificates with WebTransport by providing the certificate's SHA-256 fingerprint. The certificate must be valid for no more than 14 days.
// Connect with certificate fingerprint const transport = new WebTransport('https://localhost:4433', { serverCertificateHashes: [{ algorithm: 'sha-256', value: hexToArrayBuffer('7E:F0:20:...') }] }); // Generate fingerprint from certificate (server-side) // openssl x509 -in cert.pem -outform DER | \ // openssl dgst -sha256 -binary | \ // xxd -p -c 256 // Constraints: // - Certificate validity: max 14 days // - Must be self-signed or from a private CA // - Not applicable to publicly-trusted certificates
// Create WebTransport connection const transport = new WebTransport('https://example.com:4433/wt'); // Wait for connection await transport.ready; // === DATAGRAMS === // Send unreliable datagram const dgWriter = transport.datagrams.writable.getWriter(); await dgWriter.write(new TextEncoder().encode('ping')); // Receive datagrams const dgReader = transport.datagrams.readable.getReader(); while (true) { const { value, done } = await dgReader.read(); if (done) break; console.log('Datagram:', new TextDecoder().decode(value)); } // === UNIDIRECTIONAL STREAMS === // Send (client-initiated) const uniStream = await transport.createUnidirectionalStream(); const uniWriter = uniStream.getWriter(); await uniWriter.write(data); await uniWriter.close(); // Receive (server-initiated) const uniReader = transport.incomingUnidirectionalStreams.getReader(); while (true) { const { value: stream, done } = await uniReader.read(); if (done) break; // Read from stream.getReader() } // === BIDIRECTIONAL STREAMS === const bidiStream = await transport.createBidirectionalStream(); const bidiWriter = bidiStream.writable.getWriter(); const bidiReader = bidiStream.readable.getReader(); await bidiWriter.write(request); const { value: response } = await bidiReader.read(); // === CLEANUP === transport.close({ closeCode: 0, reason: 'Done' });
