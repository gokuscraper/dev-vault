⚙ client_mode Options

Select a launch handler client_mode to see how the PWA behaves when launched. Each mode controls whether a new window is created or an existing one is reused.

auto
User agent decides the most sensible context. Default behavior when no handler is specified.
default
navigate-new
Always opens a new browsing context (window/tab) and navigates to the launch URL.
navigate-existing
Reuses the most recently interacted-with browsing context and navigates it to the launch URL.
focus-existing
Focuses an existing context without navigating. The app handles the launch URL via LaunchQueue.

📋 LaunchQueue Consumer

Simulate LaunchQueue.setConsumer() behavior. This API lets your app receive launch parameters programmatically.

Live Demo
API Reference
--:--:-- INFO Waiting for launch events...
// Register a consumer for launch events if ('launchQueue' in window) { window.launchQueue.setConsumer(async (launchParams) => { // launchParams.targetURL - the URL that triggered the launch console.log('Launched with:', launchParams.targetURL); // launchParams.files - FileSystemHandle[] (if file handling) if (launchParams.files?.length) { for (const fileHandle of launchParams.files) { const file = await fileHandle.getFile(); console.log('File:', file.name, file.type); } } }); }

WICG Draft
The Launch Handler API is part of the PWA enhancements led by the Web Incubator Community Group (WICG). It works in conjunction with the File Handling API and URL handlers to give PWAs more control over how they respond to being launched. The launchQueue interface is shared with the File Handling API.
