titlebar-area-* CSS Environment Variables

When Window Controls Overlay is active, the browser exposes four CSS environment variables that describe the available titlebar area (the space not occupied by window controls).

env(titlebar-area-x)
0px
Left edge of the available titlebar area
env(titlebar-area-y)
0px
Top edge of the available titlebar area
env(titlebar-area-width)
100%
Width of the available titlebar area
env(titlebar-area-height)
36px
Height of the titlebar area

The titlebar above uses env(titlebar-area-*) values to position content around the window controls.

/* CSS using titlebar-area env() variables */ .custom-titlebar { position: fixed; top: env(titlebar-area-y, 0); left: env(titlebar-area-x, 0); width: env(titlebar-area-width, 100%); height: env(titlebar-area-height, 36px); /* Make it draggable */ -webkit-app-region: drag; app-region: drag; } /* Buttons inside titlebar must be non-draggable */ .custom-titlebar button { -webkit-app-region: no-drag; app-region: no-drag; }

Geometry Change Events

The navigator.windowControlsOverlay object fires a geometrychange event when the titlebar area changes (e.g., when entering/exiting fullscreen, or when WCO is toggled).

Event Log

Geometry change events will appear here.

// Listen for geometry changes if ('windowControlsOverlay' in navigator) { navigator.windowControlsOverlay.addEventListener( 'geometrychange', (event) => { const { x, y, width, height } = event.titlebarAreaRect; const visible = event.visible; console.log(`WCO visible: ${visible}`); console.log(`Area: ${x},${y} ${width}x${height}`); // Adapt layout based on available space if (visible) { document.body.classList.add('wco-active'); } else { document.body.classList.remove('wco-active'); } } ); // Check initial state const { visible } = navigator.windowControlsOverlay; console.log(`Initial WCO state: ${visible}`); }

PWA display_override Configuration

To enable Window Controls Overlay, add "window-controls-overlay" to the display_override array in your web app manifest.

manifest.json

{ "name": "My PWA with Custom Titlebar", "short_name": "MyPWA", "start_url": "/", "display": "standalone", "display_override": [ "window-controls-overlay" ], "theme_color": "#1a1a2e", "background_color": "#0a0a0a", "icons": [ { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" }, { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" } ] }

Complete Setup Checklist

  • ☑ Add "display_override": ["window-controls-overlay"] to manifest.json
  • ☑ Create a custom titlebar with env(titlebar-area-*) CSS values
  • ☑ Set app-region: drag on titlebar and app-region: no-drag on interactive elements
  • ☑ Listen for geometrychange events to adapt layout
  • ☑ Provide fallback for when WCO is not active (regular standalone mode)
  • ☑ Match theme_color to your titlebar background
