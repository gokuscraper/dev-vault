Cross-Page Transitions (MPA)

The pagereveal event fires when a page is first rendered after a cross-document navigation. Combined with @view-transition, it enables smooth MPA page transitions without JavaScript frameworks.

How It Works

1
@view-transition

Add @view-transition { navigation: auto; } to both the source and destination pages. This opts into cross-document view transitions.

2
Browser captures snapshot

Before navigation, the browser captures a screenshot of the old page. After loading the new page, it captures the new state.

3
pagereveal fires

The pagereveal event fires on the new page's window. Use it to customize the transition based on navigation context.

4
Animate pseudo-elements

::view-transition-old(root) and ::view-transition-new(root) are animated. Customize with CSS @keyframes.

View transition types let you apply different animations for different navigation patterns (e.g., forward vs back, slide vs fade). Set types via e.viewTransition.types in the pagereveal handler.

/* In pagereveal handler, set the transition type */ window.addEventListener('pagereveal', (e) => { if (!e.viewTransition) return; const from = navigation.activation?.from?.url; const to = navigation.currentEntry?.url; // Determine direction based on URL hierarchy if (isForward(from, to)) { e.viewTransition.types.add('slide-forward'); } else { e.viewTransition.types.add('slide-backward'); } }); /* Style based on type */ ::view-transition-old(root):only-child { animation: none; /* default */ } html:active-view-transition-type(slide-forward) { &::view-transition-old(root) { animation: slide-out-left 0.4s ease; } &::view-transition-new(root) { animation: slide-in-right 0.4s ease; } } html:active-view-transition-type(slide-backward) { &::view-transition-old(root) { animation: slide-out-right 0.4s ease; } &::view-transition-new(root) { animation: slide-in-left 0.4s ease; } }

@view-transition At-Rule

The @view-transition CSS at-rule opts a document into cross-document view transitions. Both the source and destination pages must include this rule.

Syntax @view-transition

/* Opt into cross-document view transitions */ @view-transition { navigation: auto; } /* Values for navigation: auto — enable for same-origin navigations none — disable (default) */ /* The at-rule must be in BOTH pages: source.html and destination.html */

Important: This only works for same-origin navigations. Cross-origin navigations cannot have view transitions for security reasons.

Tip: Place the @view-transition rule in your global stylesheet so all pages automatically participate in cross-document transitions.

Named Transitions view-transition-name

/* Assign view-transition-name to persist elements across page transitions */ /* Page 1: list page */ .hero-image { view-transition-name: hero; } .page-title { view-transition-name: title; } /* Page 2: detail page */ .detail-image { view-transition-name: hero; /* same name! */ } .detail-title { view-transition-name: title; /* same name! */ } /* Customize named element transitions */ ::view-transition-old(hero) { animation: fade-scale-out 0.4s ease; } ::view-transition-new(hero) { animation: fade-scale-in 0.4s ease; } /* RULE: view-transition-name must be unique per page. Two elements on the same page cannot share the same name. */

Performance Implications

View transitions involve screenshotting the old page and animating pseudo-elements. Understanding the performance characteristics helps you build smooth transitions.

Performance Considerations analysis

✅ Compositor-Driven

View transition animations run on the compositor thread. They're as performant as CSS transitions — no main-thread blocking.

⚠ Screenshot Cost

The browser must capture a rasterized snapshot of the old page. Complex pages with many layers may have a small capture delay.

⚠ Named Elements

Each view-transition-name creates additional pseudo-elements. Use sparingly — 5-10 named elements is fine, 100+ may cause issues.

✅ Progressive Enhancement

Non-supporting browsers simply ignore @view-transition. Navigation works normally. Zero cost for unsupported browsers.

✅ prefers-reduced-motion

Wrap transition styles in @media (prefers-reduced-motion: no-preference) to respect user accessibility settings.

❌ Cross-Origin Block

Cross-origin navigations cannot have view transitions. This is a security restriction — no workaround exists.

/* Respect user motion preferences */ @media (prefers-reduced-motion: no-preference) { @view-transition { navigation: auto; } ::view-transition-old(root) { animation: fade-out 0.3s ease; } ::view-transition-new(root) { animation: fade-in 0.3s ease; } } /* For reduced-motion users, transitions are skipped entirely — instant page swap */

Framework Patterns

Common patterns for integrating MPA view transitions with popular frameworks and static site generators.

Vanilla HTML/CSS MPA

The simplest approach. Add the at-rule to your global stylesheet and customize with pseudo-elements.

/* global.css — include on ALL pages */ @view-transition { navigation: auto; } ::view-transition-old(root) { animation: slide-out 0.3s ease; } ::view-transition-new(root) { animation: slide-in 0.3s ease; } @keyframes slide-out { to { transform: translateX(-100%); opacity: 0; } } @keyframes slide-in { from { transform: translateX(100%); opacity: 0; } } /* Named elements for shared transitions */ .logo { view-transition-name: logo; } .nav { view-transition-name: nav; }

Astro MPA

Astro has built-in view transitions support via the <ViewTransitions /> component.

--- // Layout.astro import { ViewTransitions } from 'astro:transitions'; --- <html> <head> <ViewTransitions /> </head> <body> <slot /> </body> </html> <!-- Use transition:name on elements --> <img transition:name="hero" src="..." /> <h1 transition:name="title">...</h1> <!-- Astro adds @view-transition { navigation: auto; } automatically when you use ViewTransitions -->

Next.js (App Router) SPA

Next.js uses client-side navigation, so use the SPA document.startViewTransition() API.

// For SPA view transitions in Next.js // Use the startViewTransition API in route changes 'use client'; import { useRouter } from 'next/navigation'; function NavLink({ href, children }) { const router = useRouter(); const handleClick = (e) => { e.preventDefault(); if (!document.startViewTransition) { router.push(href); return; } document.startViewTransition(() => { router.push(href); }); }; return <a href={href} onClick={handleClick}>{children}</a>; }

Eleventy / Hugo / Jekyll MPA

Static site generators output plain HTML. Just add the CSS at-rule to your base stylesheet.

/* Add to your base/global stylesheet */ @view-transition { navigation: auto; } /* Dynamic type-based transitions via pagereveal */ <script> window.addEventListener('pagereveal', (e) => { if (!e.viewTransition) return; // Add types based on page metadata const pageType = document.querySelector('meta[name="page-type"]')?.content; if (pageType) { e.viewTransition.types.add(pageType); } }); </script> /* In each page template: */ <meta name="page-type" content="article">

API Reference

API Type Description
pagereveal Window event Fires when a page is first rendered after cross-document navigation. The event has a viewTransition property if a view transition is active.
pageswap Window event Fires on the old page just before it is replaced. Use to customize exit animations based on destination URL.
@view-transition CSS at-rule Opts a document into cross-document view transitions. Required on both source and destination pages.
::view-transition-old(name) CSS pseudo-element The captured snapshot of the old page or named element. Animate this for exit effects.
::view-transition-new(name) CSS pseudo-element The live rendering of the new page or named element. Animate this for entry effects.
view-transition-name CSS property Assigns a name to an element for shared element transitions. Names must be unique per page.
e.viewTransition.types Set<string> Add type strings to apply different CSS animations via :active-view-transition-type(name).
