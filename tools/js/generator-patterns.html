Generator Basics: yield, next, return, throw

A generator function (declared with function*) returns a Generator object that conforms to the iterator protocol. Each yield pauses execution and next() resumes it.

Two-way Communication

The value passed to next(value) becomes the result of the yield expression inside the generator. This enables bidirectional data flow.
Accumulator Generator
function* accumulator() { let total = 0; while (true) { const input = yield total; total += input ?? 0; } }

Lazy Evaluation

Generators compute values on demand, not all at once. This is memory-efficient for large or expensive computations — values are only produced when requested.

Generator Pipeline

Chain generators together to create data processing pipelines. Each stage pulls values lazily from the previous stage — like Unix pipes.
Pipeline Demo
// Source → Filter → Map → Take function* range(start, end) { for (let i=start; i<=end; i++) yield i; } function* filter(gen, fn) { for (const v of gen) if (fn(v)) yield v; } function* map(gen, fn) { for (const v of gen) yield fn(v); } function* take(gen, n) { let i=0; for (const v of gen) { if (i++>=n) return; yield v; } }

AsyncIterator Protocol

An object is an async iterable if it has a [Symbol.asyncIterator]() method that returns an async iterator (an object with an async next() method).
Protocol Demo
// Custom async iterable const asyncRange = { from: 1, to: 5, [Symbol.asyncIterator]() { let current = this.from; const last = this.to; return { async next() { await new Promise(r => setTimeout(r, 200)); if (current <= last) { return { value: current++, done: false }; } return { done: true }; } }; } };
