Background Fetch Registration Flow

The Background Fetch API requires a Service Worker registration. This flow visualizes the complete lifecycle from SW registration through fetch completion.

1

Register Service Worker

navigator.serviceWorker.register('/sw.js')

2

Get SW Registration

navigator.serviceWorker.ready

3

Call backgroundFetch.fetch()

registration.backgroundFetch.fetch(id, urls, options)

4

Browser Shows Download UI

Progress notification appears in system tray / notification area

5

Track Progress

bgFetch.addEventListener('progress', ...)

6

Handle Completion

SW receives backgroundfetchsuccess or backgroundfetchfail

// Complete Background Fetch registration const swReg = await navigator.serviceWorker.register('/sw.js'); const registration = await navigator.serviceWorker.ready; // Start a background fetch const bgFetch = await registration.backgroundFetch.fetch( 'my-download-id', ['/large-file-1.zip', '/large-file-2.zip'], { title: 'Downloading resources...', icons: [{ src: '/icon.png', sizes: '192x192', type: 'image/png' }], downloadTotal: 1024 * 1024 * 50 // 50 MB expected } ); // Track progress bgFetch.addEventListener('progress', () => { const pct = bgFetch.downloaded / bgFetch.downloadTotal * 100; console.log(`Progress: ${pct.toFixed(1)}%`); });

Service Worker Code

// sw.js - Service Worker event handlers // Fired when all requests complete successfully self.addEventListener('backgroundfetchsuccess', (event) => { const bgFetch = event.registration; event.waitUntil(async function() { // Get all fetched records const records = await bgFetch.matchAll(); // Cache the responses const cache = await caches.open('bg-fetch-cache'); for (const record of records) { const response = await record.responseReady; await cache.put(record.request, response); } // Notify the user event.updateUI({ title: 'Download complete!' }); }()); }); // Fired when any request fails self.addEventListener('backgroundfetchfail', (event) => { event.updateUI({ title: 'Download failed' }); }); // Fired when user aborts via browser UI self.addEventListener('backgroundfetchabort', (event) => { console.log('User aborted:', event.registration.id); }); // Fired when user clicks the download notification self.addEventListener('backgroundfetchclick', (event) => { event.waitUntil( clients.openWindow('/downloads') ); });

Real-World Use Cases

Background Fetch is designed for downloading large resources that may take a long time. Unlike regular fetch(), downloads persist even if the user navigates away or closes the tab.

Podcast Downloads

Download episodes for offline listening. Multiple files, resume on network recovery.

Large File Transfer

Download datasets, ML models, or archive files that may take minutes to complete.

Offline Map Tiles

Pre-download map regions for offline navigation. Hundreds of small tile images.

Course Content

Download video lectures and materials for offline learning. Multi-file bundles.

Game Assets

Download textures, audio, and level data for a web game. Large combined payload.

Media Library Sync

Sync a music or photo library for offline access. Incremental downloads.

Key Limitations

  • ⚠️ Requires HTTPS (or localhost for development)
  • ⚠️ Requires an active Service Worker registration
  • ⚠️ Browser may throttle or cancel downloads on low battery
  • ⚠️ No upload support (download only)
  • ⚠️ Limited to same-origin or CORS-enabled resources
  • ⚠️ User can always abort from the browser download UI
