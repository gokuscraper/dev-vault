Custom Element Lifecycle Callbacks

Custom elements have four lifecycle callbacks that fire at specific points in the element's life.

constructor()
→
connectedCallback()
→
attributeChangedCallback()
→
disconnectedCallback()
CallbackWhenCommon Use
constructor() Element created or upgraded Init state, attach shadow DOM, set defaults
connectedCallback() Added to DOM Fetch data, start observers, render content
disconnectedCallback() Removed from DOM Cleanup listeners, cancel timers, abort fetches
attributeChangedCallback() Observed attribute changes Update rendering based on attribute values
adoptedCallback() Moved to new document Adapt to new document context (rare) 
class MyComponent extends HTMLElement {
  static observedAttributes = ['color', 'size'];

  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
    console.log('constructor');
  }

  connectedCallback() {
    console.log('connected to DOM');
    this.shadowRoot.innerHTML = '<p>Hello!</p>';
  }

  disconnectedCallback() {
    console.log('removed from DOM');
  }

  attributeChangedCallback(name, oldVal, newVal) {
    console.log(`${name}: ${oldVal} → ${newVal}`);
  }

  adoptedCallback() {
    console.log('moved to new document');
  }
}

customElements.define('my-component', MyComponent);

Observed Attributes & attributeChangedCallback

Only attributes listed in static observedAttributes trigger attributeChangedCallback.

class ColorBox extends HTMLElement {
  // Only these attributes are observed
  static observedAttributes = ['color', 'size'];

  attributeChangedCallback(name, oldValue, newValue) {
    // Fires BEFORE connectedCallback on initial parse!
    if (name === 'color') {
      this.style.background = newValue;
    }
    if (name === 'size') {
      this.style.width = newValue + 'px';
      this.style.height = newValue + 'px';
    }
  }
}
Important: attributeChangedCallback fires before connectedCallback when attributes exist in the HTML markup. Always handle the case where the element is not yet connected.

Custom Element Upgrade Process

Elements can exist in the DOM before their class is registered. When customElements.define() is called, existing elements are "upgraded" — their constructor runs and they become full custom elements.

// 1. Element exists in HTML but class not yet defined
<my-widget>I exist but I'm not upgraded yet</my-widget>

// 2. At this point, my-widget is an HTMLUnknownElement
const el = document.querySelector('my-widget');
el instanceof HTMLElement;    // true
el instanceof MyWidget;      // false (not yet!)

// 3. Define the class — upgrade happens automatically
customElements.define('my-widget', MyWidget);

// 4. Now it's upgraded
el instanceof MyWidget;      // true

// Wait for upgrade programmatically:
await customElements.whenDefined('my-widget');

// Force upgrade:
customElements.upgrade(el);

Shadow DOM Modes

Shadow DOM provides encapsulation for styles and markup. It has two modes: open and closed.

Open Mode Accessible

this.attachShadow({ mode: 'open' });
// element.shadowRoot → ShadowRoot
// External code CAN access

Closed Mode Hidden

this._shadow = this.attachShadow({
  mode: 'closed'
});
// element.shadowRoot → null
// Only internal ref can access
Best practice: Use open mode. Closed mode gives a false sense of security — it doesn't prevent determined access and makes debugging harder.

Slots & Slot Change Events

Slots allow content projection from the light DOM into the shadow DOM. Named slots target specific content; the default slot catches everything else.

// Shadow DOM template with slots
this.shadowRoot.innerHTML = `
  <style>
    ::slotted(h2) { color: blue; }
    ::slotted(*) { margin: 0.5rem 0; }
  </style>
  <header>
    <slot name="title">Default Title</slot>
  </header>
  <main>
    <slot>Default content</slot>
  </main>
  <footer>
    <slot name="actions"></slot>
  </footer>
`;

// Listen for slot changes
const slot = this.shadowRoot.querySelector('slot');
slot.addEventListener('slotchange', (e) => {
  const assigned = e.target.assignedNodes();
  console.log('Slot content changed:', assigned);
});

Form-Associated Custom Elements

Custom elements can participate in form submission by setting static formAssociated = true and using the ElementInternals API.

class FancyInput extends HTMLElement {
  static formAssociated = true;

  constructor() {
    super();
    this._internals = this.attachInternals();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() {
    this.shadowRoot.innerHTML = '<input type="text">';
    const input = this.shadowRoot.querySelector('input');
    input.addEventListener('input', (e) => {
      this._internals.setFormValue(e.target.value);
    });
  }

  // Form lifecycle callbacks
  formAssociatedCallback(form) {
    console.log('Associated with form', form);
  }

  formResetCallback() {
    this.shadowRoot.querySelector('input').value = '';
    this._internals.setFormValue('');
  }

  formDisabledCallback(disabled) {
    this.shadowRoot.querySelector('input').disabled = disabled;
  }

  formStateRestoreCallback(state, mode) {
    this.shadowRoot.querySelector('input').value = state;
  }
}

ElementInternals API

The ElementInternals interface provides custom elements with form participation, ARIA, custom states, and validation APIs.

FeatureMethods/PropertiesPurpose
Form value setFormValue(value) Set value submitted with form
Validation setValidity(flags, message) Custom validation constraints
ARIA role, ariaLabel, ... Set ARIA without attributes
Custom States states.add('checked') CSS: :state(checked)
Shadow Root shadowRoot Access even closed shadow root 
class ToggleSwitch extends HTMLElement {
  static formAssociated = true;

  constructor() {
    super();
    this._internals = this.attachInternals();
    this._checked = false;

    // ARIA role without attribute
    this._internals.role = 'switch';
    this._internals.ariaChecked = 'false';
  }

  toggle() {
    this._checked = !this._checked;
    this._internals.setFormValue(this._checked ? 'on' : 'off');
    this._internals.ariaChecked = String(this._checked);

    // Custom state for CSS :state(checked)
    if (this._checked) {
      this._internals.states.add('checked');
    } else {
      this._internals.states.delete('checked');
    }
  }
}
