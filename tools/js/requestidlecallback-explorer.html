🛠 Cooperative Scheduling Patterns

Chunk Processing Pattern

Process a large array in small chunks during idle time, yielding back to the browser between chunks.

function processInChunks(items, processFn, chunkSize = 10) { let i = 0; function processChunk(deadline) { while (i < items.length && deadline.timeRemaining() > 0) { const end = Math.min(i + chunkSize, items.length); for (let j = i; j < end; j++) processFn(items[j]); i = end; } if (i < items.length) requestIdleCallback(processChunk); } requestIdleCallback(processChunk); }

Priority Queue Pattern

Use timeout option to enforce priority. Critical tasks get short timeouts, background tasks wait for true idle.

const PRIORITY = { HIGH: 100, MEDIUM: 1000, LOW: 0 }; function scheduleTask(task, priority) { const opts = priority > 0 ? { timeout: priority } : {}; return requestIdleCallback((deadline) => { // If didTimeout, task was forced due to priority if (deadline.didTimeout) console.warn('Forced execution:', task.name); task.run(); }, opts); }

Deadline-Aware Pattern

Check timeRemaining() before each unit of work. If time is short, defer to next idle period.

function deadlineAwareWork(tasks) { function doWork(deadline) { while (tasks.length > 0 && deadline.timeRemaining() > 1) { const task = tasks.shift(); // Estimate: will this task finish in remaining time? if (task.estimatedMs > deadline.timeRemaining()) { tasks.unshift(task); // Put it back break; } task.execute(); } if (tasks.length > 0) requestIdleCallback(doWork); } requestIdleCallback(doWork); }
