Positive Lookbehind (?<=...)

Matches a pattern only if it's preceded by another pattern. The lookbehind itself is zero-width: it doesn't consume characters.

Pattern: (?<=@)\w+
// Match usernames after @ const text = "Contact @alice or @bob for help"; const re = /(?<=@)\w+/g; // Matches: "alice", "bob" // The @ is NOT included in the match
Try it
Pattern: (?<=\d{3}-)\d{4}
// Match last 4 digits of phone numbers const text = "Call 555-1234 or 888-5678"; const re = /(?<=\d{3}-)\d{4}/g; // Matches: "1234", "5678"
Try it

Negative Lookbehind (?<!...)

Matches only if NOT preceded by the given pattern.

Pattern: (?<!un)happy
Pattern: (?<!\.)(\d+) — Numbers NOT after a dot

Named Capture Groups (?<name>...)

Named groups let you reference captures by name instead of index. Access via match.groups.name.

Date Parsing
const re = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/; const m = "2025-03-12".match(re); m.groups.year // "2025" m.groups.month // "03" m.groups.day // "12"
URL Parsing
const re = /(?<proto>\w+):\/\/(?<host>[^\/]+)(?<path>\/[^\s]*)?/; const m = "https://example.com/path".match(re); m.groups.proto // "https" m.groups.host // "example.com" m.groups.path // "/path"

Backreferences with Named Groups

Use \k<name> to backreference a named group within the pattern.

Unicode Property Escapes \p{...}

Unicode property escapes match characters by their Unicode property. Requires the u flag. Categories include Letter, Number, Script, Emoji, and more.

\p{Letter}
Any letter from any script
\p{Number}
Any numeric character
\p{Script=Greek}
Greek script characters
\p{Emoji}
Emoji characters
\P{Letter}
NOT a letter (inverse)
\p{Lu}
Uppercase letter
\p{Ll}

dotAll Flag (s)

The s flag makes . match any character including \n, \r, line separator (U+2028), and paragraph separator (U+2029). Without it, . only matches non-newline characters.

Without s flag: /<div>.+<\/div>/
With s flag: /<div>.+<\/div>/s
Multiline content extraction

String.prototype.matchAll()

matchAll() returns an iterator of all matches with full group information. Unlike match() with g flag, it preserves capture groups.

// match() with g flag loses groups: "a1 b2".match(/(\w)(\d)/g) // ["a1", "b2"] — no groups! // matchAll() preserves everything: for (const m of "a1 b2".matchAll(/(\w)(\d)/g)) { // m[0] = "a1", m[1] = "a", m[2] = "1", m.index = 0 // m[0] = "b2", m[1] = "b", m[2] = "2", m.index = 3 }

v Flag (UnicodeSets)

ES2024

The v flag (replaces u) enables set notation in character classes: intersection [A&&B], subtraction [A--B], and \p{} inside classes. Also handles multi-codepoint properties.

Set Intersection: Letters AND ASCII
Set Subtraction: Digits minus 0-5
String literals in classes (multi-codepoint)
// v flag features summary: // Set intersection: [\p{Letter}&&[a-z]] — lowercase ASCII letters // Set subtraction: [\p{Decimal_Number}--[0-9]] — non-ASCII digits // String literals: [\q{ff|fi|fl}] — ligature sequences // \p inside []: [\p{ASCII}&&\p{Letter}] — intersect properties // Replaces u flag: /pattern/v (cannot combine with u)
