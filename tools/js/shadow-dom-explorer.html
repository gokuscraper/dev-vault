◈ Shadow DOM

Shadow DOM provides encapsulation for DOM and CSS. A shadow tree is attached to an element (the host) and rendered instead of, or in addition to, the host's children.

Open vs Closed

  • mode: "open" — element.shadowRoot returns the shadow root. External JS can access internals.
  • mode: "closed" — element.shadowRoot returns null. Internals are hidden (but not truly secure).

Style Encapsulation

Styles inside shadow DOM do not leak out, and document styles do not penetrate in. Exceptions:

  • ::part(name) — style elements with part="name" from outside
  • ::slotted(selector) — style slotted light DOM children from inside
  • Inherited properties (color, font) still cascade through
  • CSS custom properties pierce the shadow boundary

<slot> Distribution

Slots project light DOM children into the shadow tree. <slot> shows default content when nothing is distributed. <slot name="x"> matches slot="x" on light DOM children.

::slotted()

Used inside shadow CSS to style elements distributed through <slot>. Only targets top-level slotted elements, not their descendants. Specificity is lower than light DOM styles.

::part()

Exposes named parts of shadow DOM to external styling via part="name" attribute. The host author controls which parts are exposed. Used from the document level: my-element::part(header) { ... }.

adoptedStyleSheets

A more performant way to share styles. Create a CSSStyleSheet, call .replaceSync(css), and assign to shadowRoot.adoptedStyleSheets. The same sheet object can be shared across multiple shadow roots.
