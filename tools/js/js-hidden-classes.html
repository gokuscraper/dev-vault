What Are Hidden Classes?

V8 (and other JS engines) create internal "hidden classes" (also called Shapes or Maps) to track the structure of objects. Objects with the same properties in the same order share the same hidden class, enabling fast property access via offsets instead of hash lookups.

Transition Chains

When you add a property to an object, V8 creates a new hidden class and links it to the previous one. This forms a transition chain:

  • {} → HC0 (empty)
  • obj.x = 1 → HC1 (has "x" at offset 0)
  • obj.y = 2 → HC2 (has "x" at 0, "y" at 1)

If another object adds the same properties in the same order, it reuses the existing transition chain — no new hidden classes needed.

Inline Caching States

  • Monomorphic — All objects at a call site share one hidden class. Fastest — V8 generates optimized machine code.
  • Polymorphic — 2-4 different hidden classes seen. V8 checks a small list. Slower but still optimized.
  • Megamorphic — 5+ hidden classes. V8 falls back to dictionary lookup. Slowest — deoptimization likely.

Performance Rules

  • Always initialize properties in the same order — Different orders create different hidden classes
  • Initialize all properties in the constructor — Don't add properties later
  • Don't delete properties — delete obj.x forces V8 to use slower dictionary mode
  • Don't mix types — Changing obj.x from number to string changes the hidden class
  • Use classes/constructor functions — They naturally produce consistent object shapes

Real-World Impact

Hidden class mismatches are one of the most common causes of unexpected deoptimization. In tight loops, the difference between monomorphic and megamorphic property access can be 10-100x.
