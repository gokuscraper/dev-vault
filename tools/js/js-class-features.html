Private Fields (#field)

Private fields are truly private — not accessible outside the class body. They use the # prefix and are checked at the syntax level.

class BankAccount {
  #balance = 0;         // private field with default
  #owner;                // private field (undefined)

  constructor(owner, initial) {
    this.#owner = owner;
    this.#balance = initial;
  }

  deposit(amount) {
    if (amount <= 0) throw new Error('Invalid amount');
    this.#balance += amount;
    return this.#balance;
  }

  get balance() {
    return this.#balance;
  }
}

const acct = new BankAccount('Alice', 100);
acct.balance;     // 100 (via getter)
acct.#balance;    // SyntaxError: Private field
acct['#balance']; // undefined (it's not a string key!)
Key difference from conventions: Unlike the _underscore convention, #private fields are enforced by the engine. They cannot be accessed via reflection, proxies, or bracket notation.

Private Methods (#method())

Methods can also be private. They follow the same rules as private fields — only accessible within the class body.

class Validator {
  #rules = [];

  addRule(rule) {
    this.#rules.push(rule);
  }

  // Private method
  #runRule(rule, value) {
    return rule.test ? rule.test(value) : rule(value);
  }

  validate(value) {
    return this.#rules.every(rule => this.#runRule(rule, value));
  }

  // Private getter
  get #ruleCount() {
    return this.#rules.length;
  }

  // Private static method
  static #isValid(v) {
    return v !== null && v !== undefined;
  }
}

Types of Private Members

TypeSyntaxAccess

Static Initialization Blocks

Static blocks run once when the class is evaluated. They have access to private static members and can perform complex initialization.

class Config {
  static #envMap;
  static debug;
  static version;

  // Static initialization block
  static {
    // Complex initialization logic
    try {
      this.#envMap = new Map([
        ['dev', { debug: true, api: 'localhost' }],
        ['prod', { debug: false, api: 'api.example.com' }],
      ]);
      const env = 'dev';
      const config = this.#envMap.get(env);
      this.debug = config.debug;
      this.version = '1.0.0';
    } catch (e) {
      this.debug = false;
      this.version = 'unknown';
    }
  }
}

// Can also share private access with external code:
let getPrivate;
class Secret {
  #value = 42;
  static {
    getPrivate = (obj) => obj.#value;
  }
}
getPrivate(new Secret()); // 42 — "friend" access pattern

Class Fields (Public & Private)

Class fields allow you to declare properties directly in the class body, with or without default values.

class User {
  // Public fields
  name = 'anonymous';
  role = 'user';
  createdAt = new Date();

  // Private fields
  #email;
  #loginCount = 0;

  // Static fields
  static totalUsers = 0;
  static #instances = [];

  constructor(name, email) {
    this.name = name;
    this.#email = email;
    User.totalUsers++;
    User.#instances.push(this);
  }
}

// Each instance gets its own copy of fields
const u1 = new User('Alice', 'alice@example.com');
const u2 = new User('Bob', 'bob@example.com');
u1.createdAt === u2.createdAt; // false (new Date() per instance)

Old Pattern (constructor only)

class Old {
  constructor() {
    this.x = 0;
    this.y = 0;
    this._private = [];
  }
}

New Pattern (class fields)

class New {
  x = 0;
  y = 0;
  #private = [];
}

Auto-Accessor (accessor keyword)

The accessor keyword creates a field with automatic getter and setter, backed by a private storage slot.

class ReactiveElement {
  // Auto-accessor: creates getter + setter + private backing field
  accessor name = 'default';
  accessor count = 0;

  // Equivalent manual code:
  // #name = 'default';
  // get name() { return this.#name; }
  // set name(v) { this.#name = v; }
}

const el = new ReactiveElement();
el.name;           // 'default' (via auto getter)
el.name = 'test'; // (via auto setter)
Why auto-accessor? It's the foundation for decorators. A decorator can intercept the getter/setter to add reactivity, validation, logging, etc. Without accessor, decorators can only wrap methods, not fields. 
// With a decorator (Stage 3)
function logged(value, context) {
  return {
    get() {
      console.log(`Getting ${context.name}`);
      return value.get.call(this);
    },
    set(v) {
      console.log(`Setting ${context.name} = ${v}`);
      value.set.call(this, v);
    }
  };
}

class Model {
  @logged accessor title = '';
}

Object.hasOwn() vs hasOwnProperty()

Object.hasOwn() is the modern replacement for obj.hasOwnProperty(). It's safer and simpler.

Old: hasOwnProperty() Fragile

// Can be shadowed!
const obj = {
  hasOwnProperty: () => false,
  key: 'value'
};
obj.hasOwnProperty('key'); // false! (wrong)

// Fails on null prototype
const dict = Object.create(null);
dict.key = 'value';
dict.hasOwnProperty('key'); // TypeError!

// Safe but verbose
Object.prototype.hasOwnProperty
  .call(obj, 'key');

New: Object.hasOwn() Safe

// Cannot be shadowed
const obj = {
  hasOwnProperty: () => false,
  key: 'value'
};
Object.hasOwn(obj, 'key'); // true ✓

// Works with null prototype
const dict = Object.create(null);
dict.key = 'value';
Object.hasOwn(dict, 'key'); // true ✓

// Simple and safe
Object.hasOwn(obj, 'key');

Private Field Brand Check (#field in obj)

The in operator can check if an object has a specific private field. This is called a "brand check" — it verifies the object was created by the class.

class Color {
  #red; #green; #blue;

  constructor(r, g, b) {
    this.#red = r;
    this.#green = g;
    this.#blue = b;
  }

  // Brand check: is this really a Color instance?
  static isColor(obj) {
    return #red in obj;
  }

  equals(other) {
    // Safe comparison — verify other is a Color first
    if (!(#red in other)) return false;
    return this.#red === other.#red
        && this.#green === other.#green
        && this.#blue === other.#blue;
  }
}

Color.isColor(new Color(255, 0, 0)); // true
Color.isColor({ red: 255 });           // false
Color.isColor(42);                     // false (no error!)
vs instanceof: instanceof checks the prototype chain and can be fooled with Symbol.hasInstance. The #field in obj check is unforgeable — it verifies the private slot actually exists.

Inheritance with Private Members

Private fields are not inherited by subclasses. Each class owns its own private fields — even subclasses cannot access parent's private fields.

class Animal {
  #name;
  #sound;

  constructor(name, sound) {
    this.#name = name;
    this.#sound = sound;
  }

  speak() {
    return `${this.#name} says ${this.#sound}`;
  }
}

class Dog extends Animal {
  #tricks = [];     // Dog's own private field

  constructor(name) {
    super(name, 'Woof');
  }

  learn(trick) {
    this.#tricks.push(trick);
  }

  showTricks() {
    // Can access #tricks (own) but NOT #name or #sound (parent's)
    return this.#tricks.join(', ');
  }
}

const dog = new Dog('Rex');
dog.speak();      // "Rex says Woof" (via inherited method)
dog.learn('sit');
dog.showTricks(); // "sit"
Key rule: Private fields belong to the class that declares them. A subclass method cannot read #name from the parent — only the parent's methods can. Use protected-like patterns with accessors if subclasses need access.
