Preset Scenarios

What Creates a Stacking Context?

  • Root element (<html>)
  • position: absolute/relative with z-index other than auto
  • position: fixed or sticky
  • opacity less than 1
  • transform other than none
  • filter other than none
  • isolation: isolate
  • mix-blend-mode other than normal
  • will-change with certain values
  • contain: layout/paint/strict/content
  • Flex/Grid child with z-index other than auto

Paint Order Within a Stacking Context

  1. Background and borders of the stacking context element
  2. Child stacking contexts with negative z-index
  3. In-flow, non-positioned block-level elements
  4. Non-positioned floated elements
  5. In-flow, non-positioned inline elements
  6. Child stacking contexts with z-index: 0 / auto
  7. Child stacking contexts with positive z-index

Common Gotchas

  • z-index only works on positioned elements — z-index: 999 on a position: static element does nothing
  • z-index is relative to the parent stacking context — an element with z-index: 9999 can still be below an element with z-index: 1 if they're in different stacking contexts
  • opacity < 1 creates a stacking context — this is a very common surprise. Adding opacity: 0.99 changes paint order
  • transform creates a stacking context — even transform: translateZ(0) or will-change: transform
  • Fixed positioning inside a transformed parent — position: fixed becomes relative to the transform parent, not the viewport
