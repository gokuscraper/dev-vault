⚖ EyeDropper vs Canvas getImageData

How does the EyeDropper API compare to the traditional canvas-based color picking approach?

FeatureEyeDropper APICanvas getImageData
Scope Any pixel on screen Only within canvas element
Cross-origin Works across windows/apps Tainted canvas security error
User interaction System-level color picker UI Custom implementation needed
Color space Returns sRGB hex Returns RGBA values (0-255)
Precision 8-bit per channel (hex) 8-bit per channel (Uint8)
Alpha channel Not available Available (RGBA)
Performance Single pick, no overhead Requires canvas rendering first
Browser support Chromium only All browsers

📝 Integration Examples

Code examples for using the EyeDropper API in your applications.

Basic Usage
CSS Custom Props
AbortController
// Basic EyeDropper usage async function pickColor() { if (!('EyeDropper' in window)) { alert('EyeDropper API not supported'); return; } const dropper = new EyeDropper(); try { const result = await dropper.open(); console.log(result.sRGBHex); // e.g., "#ff6b9d" document.body.style.backgroundColor = result.sRGBHex; } catch (err) { // User pressed Escape or error occurred console.log('Picker cancelled'); } }
// Set CSS custom properties from picked colors const colorVars = ['primary', 'secondary', 'accent']; let currentVar = 0; async function pickThemeColor() { const dropper = new EyeDropper(); const result = await dropper.open(); const varName = colorVars[currentVar % colorVars.length]; document.documentElement.style.setProperty( `--color-${varName}`, result.sRGBHex ); currentVar++; console.log(`Set --color-${varName} to ${result.sRGBHex}`); } // CSS usage: // :root { --color-primary: #ff6b9d; } // .header { background: var(--color-primary); }
// Using AbortController to cancel the dropper let abortController; async function startPicking() { abortController = new AbortController(); const dropper = new EyeDropper(); try { const result = await dropper.open({ signal: abortController.signal }); console.log('Picked:', result.sRGBHex); } catch (err) { if (err.name === 'AbortError') { console.log('Picker was cancelled programmatically'); } } } function cancelPicking() { abortController?.abort(); }

Requirements
Secure Context + User Gesture
The EyeDropper API is a Chromium-only feature. For cross-browser color picking, use the traditional <input type="color"> element or a canvas-based approach. The API requires a secure context (HTTPS) and must be called from a user gesture (click/tap).
