strict-dynamic Explained

When 'strict-dynamic' is present in script-src, it tells the browser to trust scripts loaded by already-trusted scripts (via nonce or hash), regardless of the allowlist. Host-based allowlists and 'self' are ignored. 

script-src 'nonce-abc123' 'strict-dynamic' https://cdn.example.com 'self';

/* With strict-dynamic:
   - 'nonce-abc123' → trusted (nonce matches)
   - Scripts loaded BY nonce'd scripts → trusted
   - https://cdn.example.com → IGNORED (host allowlist)
   - 'self' → IGNORED
   - 'unsafe-inline' → IGNORED (if nonce present)
*/

Nonce & Hash Migration Guide

Migrate from 'unsafe-inline' to nonces or hashes for stronger security.

Step 1: Choose Your Approach

Nonce-based (recommended)
Generate a unique nonce per response. Add it to the CSP header and each inline script/style tag.
// Server generates per-request
Content-Security-Policy: script-src 'nonce-abc123'

// HTML
<script nonce="abc123">
  console.log("Allowed!");
</script>
Hash-based
Pre-compute SHA hashes of inline scripts. No server-side nonce generation needed, but scripts cannot change.
// CSP header with hash
Content-Security-Policy: script-src 'sha256-base64hash...'

// HTML (exact content must match hash)
<script>
  console.log("Allowed!");
</script>

Step 3: Server Implementation Examples

      
// Node.js / Express
const crypto = require('crypto');

app.use((req, res, next) => {
  const nonce = crypto.randomBytes(16).toString('base64');
  res.locals.nonce = nonce;
  res.setHeader('Content-Security-Policy',
    `script-src 'nonce-${nonce}' 'strict-dynamic'; style-src 'self' 'nonce-${nonce}';`
  );
  next();
});

// In template: <script nonce="<%= nonce %>">...</script>


      
// Python / Django (middleware)
import secrets, base64

class CSPMiddleware:
    def __call__(self, request):
        nonce = base64.b64encode(secrets.token_bytes(16)).decode()
        request.csp_nonce = nonce
        response = self.get_response(request)
        response['Content-Security-Policy'] = (
            f"script-src 'nonce-{nonce}' 'strict-dynamic'; "
            f"style-src 'self' 'nonce-{nonce}';"
        )
        return response

    

  


  
  

    

      
CSP Reporting Configuration


      
report-uri (deprecated, but widely supported)

      
Content-Security-Policy: default-src 'self';
  report-uri https://your-domain.com/csp-report;


      
report-to (CSP Level 3)

      
// Step 1: Set the Reporting-Endpoints header
Reporting-Endpoints: csp-endpoint="https://your-domain.com/csp-report"

// Step 2: Reference in CSP
Content-Security-Policy: default-src 'self';
  report-to csp-endpoint;


      
Report-Only Mode (testing without enforcement)

      
// Reports violations but does NOT block content
Content-Security-Policy-Report-Only: default-src 'self';
  script-src 'self';
  report-to csp-endpoint;
