Set-Cookie Header Generator

Set-Cookie: name=value
Cookie Attributes:
Path - URL path scope. "/" means all pages.
Domain - Domain scope. Omit for exact host match.
Max-Age - Lifetime in seconds from now.
Expires - Absolute expiry date (UTC).
Secure - Only sent over HTTPS.
HttpOnly - Not accessible via JavaScript. Protects against XSS.
SameSite - Controls cross-site request behavior.

SameSite Comparison

Strict

Cookie is never sent with cross-site requests. Not even when following a link to the target site.

Use for: Banking, admin panels, sensitive actions

Drawback: User must re-login when arriving from external link

Lax (Default)

Cookie sent with top-level navigations (clicking links) but not with cross-site subrequests (images, iframes, AJAX).

Use for: Session cookies, general auth

Best balance of security and usability

None

Cookie sent with all requests, including cross-site. Requires Secure flag.

Use for: Third-party widgets, embedded content, cross-site APIs

Risk: CSRF vulnerability if not handled properly

Third-Party Cookie Blocking

Current Browser Landscape

  • Safari (ITP): Blocks all third-party cookies by default since 2020. 7-day cap on first-party cookies set via JS.
  • Firefox (ETP): Blocks known tracking cookies. Total Cookie Protection isolates cookies per site.
  • Chrome: Third-party cookie phase-out planned (repeatedly delayed). Partitioned cookies (CHIPS) available as alternative.
  • Edge: Follows Chromium timeline. Tracking Prevention blocks known trackers.

Alternatives to Third-Party Cookies

  • Partitioned cookies (CHIPS): Set-Cookie with Partitioned attribute. Cookie is keyed per top-level site.
  • Storage Access API: Embedded content can request access to its first-party cookies.
  • Related Website Sets: Declare related domains that can share cookies.
  • Topics API / Attribution Reporting: Privacy-preserving alternatives for ads.
