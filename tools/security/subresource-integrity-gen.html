SRI Reference

How SRI Works

  1. You generate a cryptographic hash of the resource content
  2. You add the hash as an integrity attribute on the script/link tag
  3. The browser fetches the resource and computes its hash
  4. If hashes match, the resource loads. If not, it's blocked
  5. This protects against CDN compromise, MITM attacks, and supply chain attacks

Syntax

<script src="https://cdn.example.com/lib.js"
        integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/uxy9rx7HNQlGYl1kPzQho1wx4JwY8w"
        crossorigin="anonymous"></script>

<link rel="stylesheet"
      href="https://cdn.example.com/styles.css"
      integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/uxy9rx7HNQlGYl1kPzQho1wx4JwY8w"
      crossorigin="anonymous">

Hash Algorithms

Algorithm Hash Length Security Recommendation
SHA-256 44 chars (base64) Good Minimum acceptable
SHA-384 64 chars (base64) Better Recommended
SHA-512 88 chars (base64) Best For maximum security

Crossorigin Attribute

Value When to Use
anonymous Default. Use for public CDN resources. No credentials sent.
use-credentials When the resource requires authentication (cookies, client certs).
(omitted) Same-origin resources only. SRI won't work cross-origin without it.

Browser Support

SRI is supported in all modern browsers: Chrome 45+, Firefox 43+, Safari 11+, Edge 17+. IE does not support SRI but safely ignores the integrity attribute.
