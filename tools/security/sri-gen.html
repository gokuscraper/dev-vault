What is SRI?

Subresource Integrity (SRI) lets browsers verify that fetched resources haven't been tampered with. Add an integrity attribute to <script> or <link> tags:

<script src="https://cdn.example.com/lib.js" integrity="sha384-..." crossorigin="anonymous"></script>

If the file changes (CDN compromise, version mismatch), the browser blocks it. Always include crossorigin="anonymous" for cross-origin resources.
