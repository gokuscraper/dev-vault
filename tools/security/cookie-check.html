Cookie Security Attributes

Secure — Only sent over HTTPS. Prevents interception on insecure networks.

HttpOnly — Not accessible via JavaScript (document.cookie). Prevents XSS cookie theft.

SameSite=Strict — Never sent on cross-site requests. Maximum CSRF protection.

SameSite=Lax — Sent on top-level navigations (links) but not on subresource requests (images, iframes, AJAX). Good default.

SameSite=None — Sent on all cross-site requests. Requires Secure flag. Used for third-party cookies.

Expiration

Max-Age=N — Cookie expires N seconds from now. Preferred over Expires.

Expires=date — Cookie expires at a specific date/time (HTTP date format).

If neither is set, the cookie is a session cookie — deleted when the browser closes.

Domain & Path

Domain=.example.com — Cookie available to example.com and all subdomains. Omitting Domain restricts to the exact host (more secure).

Path=/ — Cookie available on all paths. Path=/admin restricts to /admin and below.

Best Practices

1. Always use Secure and HttpOnly for session cookies.

2. Use SameSite=Lax or Strict to prevent CSRF attacks.

3. Set short Max-Age — only as long as needed.

4. Use __Host- prefix for the strictest security (requires Secure, no Domain, Path=/).

5. Use __Secure- prefix to enforce the Secure flag.
