Block and Inline Axes

Logical properties use block (perpendicular to text flow) and inline (parallel to text flow) instead of physical directions (top/right/bottom/left).

Applied Styles

/* Logical properties work with ANY writing mode */
.card {
  margin-block: 1rem;      /* top/bottom in horizontal, left/right in vertical */
  padding-inline: 1.5rem;  /* left/right in horizontal, top/bottom in vertical */
  border-block-start: 3px solid blue;  /* top in horizontal, right in vertical-rl */
  inline-size: 300px;     /* width in horizontal, height in vertical */
}

Direction (LTR/RTL) with Logical Properties

Logical properties eliminate the need for direction-specific CSS overrides. What was margin-left in LTR becomes margin-right in RTL, but margin-inline-start handles both.

Logical (adapts) Correct

Physical (breaks) Broken in RTL

Practical benefit: With logical properties, you write CSS once and it works for both LTR and RTL layouts. No need for [dir="rtl"] overrides or separate stylesheets.

Float, Clear & Resize (Logical Equivalents)

Float and clear now accept logical values inline-start and inline-end.

/* Physical (old) */
.sidebar { float: left; }
.clearfix { clear: both; }

/* Logical (new) */
.sidebar { float: inline-start; }
.clearfix { clear: inline-end; }

/* Resize */
.resizable { resize: block; }  /* vertical in horizontal-tb */
.resizable { resize: inline; } /* horizontal in horizontal-tb */

Physical to Logical Migration Checklist

A step-by-step guide for migrating your CSS from physical to logical properties.

  • 1. Replace width/height with inline-size/block-size
  • 2. Replace margin-top/bottom with margin-block-start/end or margin-block
  • 3. Replace margin-left/right with margin-inline-start/end or margin-inline
  • 4. Same for padding-* properties
  • 5. Replace top/right/bottom/left with inset-block-start/end and inset-inline-start/end
  • 6. Replace border-top/right/bottom/left with border-block/inline-start/end
  • 7. Replace border-*-radius with border-start/end-start/end-radius
  • 8. Replace text-align: left/right with text-align: start/end
  • 9. Replace float: left/right with float: inline-start/end
  • 10. Replace clear: left/right with clear: inline-start/end
  • 11. Replace resize: horizontal/vertical with resize: inline/block
  • 12. Use overflow-inline/overflow-block instead of overflow-x/overflow-y
  • 13. Consider contain-intrinsic-inline-size/block-size for content-visibility
Browser support: All logical properties listed above are supported in all modern browsers (Chrome 87+, Firefox 66+, Safari 14.1+, Edge 87+). Safe for production use.
