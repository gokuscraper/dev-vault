01 — Fundamentals

Basic @mixin and @apply

Define a reusable block of styles with @mixin, then inject them anywhere with @apply. The mixin name must be a dashed-ident (starting with --).

ⓘ
CSS mixins use the @mixin --name { ... } syntax to define, and @apply --name to use. The double-dash prefix follows CSS custom property naming conventions.
Definition — @mixin
/* Define a reusable style block */
@mixin --visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
Usage — @apply
/* Apply the mixin to any selector */
.sr-only {
  @apply --visually-hidden;
}

.skip-link:not(:focus) {
  @apply --visually-hidden;
}

/* Each gets all 8 properties injected */

Rendered Result

The .sr-only element below is visually hidden but accessible to screen readers:

This text is visually hidden

The hidden text is present in the DOM (inspect to verify) but not visible. Screen readers will announce it.

How It Works

@mixin declares a named block of declarations. Unlike custom properties, mixins can hold multiple properties, nested rules, and (eventually) parameters.

@apply expands the mixin inline, as if you'd written all its declarations at that point. It is resolved at parse time, not runtime — no cascade interaction.

02 — Parameters

Mixins with Parameters

Pass values to mixins using parameter syntax. Parameters are dashed-idents that act as local custom properties within the mixin body.

Parameterized Mixin
@mixin --card(--radius, --shadow) {
  border-radius: var(--radius, 12px);
  box-shadow: var(--shadow, 0 4px 12px rgba(0,0,0,0.1));
  padding: 20px;
  background: #fff;
  transition: box-shadow 0.3s;
}

@mixin --flex-center(--direction) {
  display: flex;
  align-items: center;
  justify-content: center;
  flex-direction: var(--direction, row);
}
Applying with Arguments
/* Use defaults */
.product-card {
  @apply --card;
}

/* Override parameters */
.hero-card {
  @apply --card(
    --radius: 24px,
    --shadow: 0 12px 40px rgba(0,0,0,0.2)
  );
}

.centered-column {
  @apply --flex-center(
    --direction: column
  );
}

Parameter Defaults

Parameters use var(--param, fallback) for defaults. When @apply is called without arguments, the fallback values are used. This mirrors how custom properties already work, but scoped to the mixin.

03 — Conditionals

Conditional Styles with @when

Use conditional logic inside mixins to apply different styles based on parameter values. The proposed @when rule enables branching within mixin bodies.

⚠
Conditional logic in CSS mixins (@when/@else) is part of the broader CSS Conditional Rules proposal. Syntax and support may evolve. The examples below show the proposed direction.
Conditional Mixin
@mixin --button-variant(--variant) {
  padding: 10px 20px;
  border-radius: 8px;
  font-weight: 600;
  cursor: pointer;
  border: none;

  @when (--variant: primary) {
    background: #6366f1;
    color: #fff;
  }
  @else when (--variant: danger) {
    background: #ef4444;
    color: #fff;
  }
  @else {
    background: transparent;
    color: #94a3b8;
    border: 1px solid #334155;
  }
}
Using Conditional Mixin
.btn-submit {
  @apply --button-variant(
    --variant: primary
  );
}

.btn-delete {
  @apply --button-variant(
    --variant: danger
  );
}

.btn-cancel {
  @apply --button-variant;
  /* Falls through to @else (ghost) */
}

Preview: Button Variants from One Mixin

Why Conditionals Matter

Without conditionals, you need a separate mixin for each variant. With @when, a single mixin can handle multiple variants, reducing code duplication. This is one of the most requested features from the Sass world, where @if/@else is heavily used.

08 — Migration

  
Sass to CSS @mixin Migration

  
Step-by-step guide to converting your Sass mixins to native CSS syntax.






  
Migration Checklist

  

    

      1.
      Rename @mixin name to @mixin --name (add double-dash prefix)
    

    

      2.
      Replace $param with --param in parameter lists
    

    

      3.
      Replace $param usage in body with var(--param, default)
    

    

      4.
      Replace @include name with @apply --name
    

    

      5.
      Replace @if/@else with @when/@else (when supported)
    

    

      6.
      Remove @content blocks (use CSS nesting instead)
    

    

      7.
      Replace @for/@each loops with repeated rules or custom properties
    

  





Example Migrations





  

    
Sass (Before)

    
@mixin flex-center($gap: 8px) {
  display: flex;
  align-items: center;
  justify-content: center;
  gap: $gap;
}

.hero {
  @include flex-center(16px);
}

  

  
→

  

    
CSS @mixin (After)

    
@mixin --flex-center(--gap) {
  display: flex;
  align-items: center;
  justify-content: center;
  gap: var(--gap, 8px);
}

.hero {
  @apply --flex-center(
    --gap: 16px
  );
}

  







  

    
Sass with @if (Before)

    
@mixin text-style($type: body) {
  @if $type == heading {
    font-size: 2rem;
    font-weight: 700;
  } @else if $type == caption {
    font-size: 0.75rem;
    color: #999;
  } @else {
    font-size: 1rem;
    line-height: 1.6;
  }
}

  

  
→

  

    
CSS @mixin with @when (After)

    
@mixin --text-style(--type) {
  @when (--type: heading) {
    font-size: 2rem;
    font-weight: 700;
  }
  @else when (--type: caption) {
    font-size: 0.75rem;
    color: #999;
  }
  @else {
    font-size: 1rem;
    line-height: 1.6;
  }
}

  






  ⚠
  
Not everything migrates 1:1. Sass features like @for, @each, string interpolation (#{}), and @content blocks don't have direct CSS mixin equivalents. These require architectural rethinking — often using CSS custom properties, nesting, or repeated declarations.
