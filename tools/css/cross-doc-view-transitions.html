How It Works

In real MPA transitions, navigating from page-a.html to page-b.html triggers the browser to:
  1. Capture a snapshot of the old page (static image)
  2. Load and render the new page as live DOM
  3. Match elements by view-transition-name
  4. Animate ::view-transition-old out, ::view-transition-new in
  5. Named elements morph (position + size) between pages

Same-Document vs Cross-Document Transitions

Compare the two approaches side by side. Both use the same underlying pseudo-element tree, but trigger differently.

SPA Same-Document

Triggered by document.startViewTransition()

MPA Cross-Document

Triggered by @view-transition { navigation: auto; }

Key Differences

SPA (Same-Document) MPA (Cross-Document)
Trigger document.startViewTransition(cb) Automatic on same-origin navigation
Opt-in JS call @view-transition { navigation: auto; }
DOM Update Developer-controlled callback Browser loads new page
Events ViewTransition promises pageswap / pagereveal
Constraint None Same-origin only
Min JS Required Yes (API call) No (CSS-only possible)
Browser Support Chrome, Edge, Safari, Firefox Chrome, Edge, Safari
