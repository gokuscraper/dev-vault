content-visibility: auto | hidden | visible

content-visibility tells the browser whether to render an element's contents. With auto, off-screen content is skipped entirely, reducing initial render cost.

contain-intrinsic-size

When content-visibility: auto skips rendering, the element has zero height by default. contain-intrinsic-size provides a placeholder size to prevent layout shifts and enable correct scrollbar sizing.

No contain-intrinsic-size: Without a size hint, skipped elements collapse to 0px height. The scrollbar jumps erratically as items enter/leave the viewport. Watch the scrollbar behavior as you scroll.

The auto keyword

contain-intrinsic-size: auto 80px uses 80px as the initial placeholder, but once the element has been rendered, the browser remembers its actual size and uses that instead. This gives the best of both worlds: correct initial scrollbar + no layout shift after scrolling.

/* Best practice: use 'auto' + a reasonable fallback */ .card { content-visibility: auto; contain-intrinsic-size: auto 200px; } /* Width and height separately */ .wide-card { content-visibility: auto; contain-intrinsic-width: auto 300px; contain-intrinsic-height: auto 150px; }

Gotchas & Common Issues

content-visibility is powerful but has edge cases you need to handle.

Warning Find-in-Page (Ctrl+F)

With content-visibility: hidden, content is completely invisible to find-in-page. With content-visibility: auto, browsers do search through hidden content and will reveal matching items — but this was not always the case in older browser versions.

Critical Accessibility

content-visibility: hidden removes content from the accessibility tree entirely — screen readers will not find it. Use auto instead for content that should remain accessible. The auto value preserves accessibility: the content is still in the accessibility tree even when rendering is skipped.

/* GOOD: accessible but performance-optimized */ .offscreen-section { content-visibility: auto; contain-intrinsic-size: auto 500px; } /* BAD for content: removes from a11y tree */ .offscreen-section { content-visibility: hidden; /* only for truly hidden UI */ }

Warning Anchor Links

Jumping to an anchor (#section-id) inside an element with content-visibility: auto works in modern browsers — the browser will render the target element and scroll to it. However, the scroll position may be slightly off if contain-intrinsic-size doesn't match the actual content height.

Note Animations & Transitions

CSS animations and transitions inside elements with content-visibility: auto are paused when the element is off-screen. They resume when the element enters the viewport. This is usually desirable (saves CPU), but can cause visual jumps if the animation state matters.

Note JavaScript Measurements

Calling getBoundingClientRect() or offsetHeight on a child of a content-visibility: hidden element returns 0. With auto, it forces the browser to render that element first (defeating the optimization for that element).

Copy-Paste Code Patterns

Ready-to-use patterns for common use cases.

Long Lists / Feeds

/* Apply to each list item or article card */ .feed-item { content-visibility: auto; contain-intrinsic-size: auto 120px; } /* The 'auto' keyword remembers actual size after first render, preventing layout shifts on subsequent visits */

Tab Panels

/* Hidden tabs stay in DOM but skip rendering entirely */ .tab-panel { content-visibility: hidden; } .tab-panel.active { content-visibility: visible; } /* Unlike display:none, this preserves rendered state. Switching back to a tab is instant — no re-render needed. Unlike visibility:hidden, this actually skips rendering work. */

Another tab. Switching is instant because the content is already rendered and cached by the browser.

Accordions / Collapsibles

/* Accordion content: hidden but preserves state */ .accordion-body { content-visibility: hidden; max-height: 0; overflow: hidden; transition: max-height 0.3s ease; } .accordion-body.open { content-visibility: visible; max-height: 500px; } /* Key benefit over display:none: form inputs, scroll positions, and component state are preserved */

Below-the-Fold Sections

/* Entire page sections that start off-screen */ .page-section { content-visibility: auto; contain-intrinsic-size: auto 800px; } /* Blog post with many sections */ article > section { content-visibility: auto; contain-intrinsic-size: auto 400px; }

Data Tables

/* Apply to table rows for large data tables */ tbody tr { content-visibility: auto; contain-intrinsic-size: auto 40px; } /* Note: works best when rows have consistent heights. Variable heights may cause scrollbar jitter. */

Virtual Scrolling Alternative

/* content-visibility: auto is a CSS-only alternative to JavaScript virtual scrolling for many use cases. Virtual scroll: removes DOM nodes entirely content-visibility: keeps DOM, skips rendering Trade-offs: - CV: simpler, works with find-in-page, no JS needed - Virtual: better for 100K+ items (DOM size matters) - CV: no flickering on fast scroll - Virtual: smaller memory footprint for extreme lists */ .virtual-scroll-alternative { content-visibility: auto; contain-intrinsic-size: auto 60px; }
