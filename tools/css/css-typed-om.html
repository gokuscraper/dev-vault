    Type Hierarchy
    
      
    CSS Typed OM organizes values into a class hierarchy:
    
      
    CSSStyleValue                   // Base class
  CSSNumericValue              // Numbers with units
    CSSUnitValue               // Single value: 16px, 2em, 45deg
    CSSMathValue               // calc(), min(), max(), clamp()
      CSSMathSum               // a + b
      CSSMathProduct           // a * b
      CSSMathMin / CSSMathMax  // min(a,b) / max(a,b)
  CSSKeywordValue              // Keywords: auto, inherit, flex
  CSSTransformValue            // Transform lists
  CSSUnparsedValue             // Custom properties, var()
  CSSImageValue                // url(), gradients

    Code Comparison
    
      
    
        
    
          
    String-based (Traditional)
    
          
    // Setting a style with strings
const el = document.querySelector('.box');

// Write
el.style.width = '100px';
el.style.opacity = '0.5';
el.style.transform = 'rotate(45deg)';

// Read — always returns a string
const w = getComputedStyle(el).width;
// "100px" — need to parseFloat()
const n = parseFloat(w); // 100

// Modify — string concatenation
el.style.width = (n + 20) + 'px';
    
        
    
        
    
          
    Typed OM (Modern)
    
          
    // Setting a style with typed values
const el = document.querySelector('.box');

// Write
el.attributeStyleMap.set('width', CSS.px(100));
el.attributeStyleMap.set('opacity', CSS.number(0.5));
el.attributeStyleMap.set('transform',
  new CSSTransformValue([
    new CSSRotate(CSS.deg(45))
  ]));

// Read — returns typed CSSUnitValue
const w = el.computedStyleMap().get('width');
// CSSUnitValue {value: 100, unit: "px"}
const n = w.value; // 100 (already a number!)

// Modify — arithmetic on typed values
el.attributeStyleMap.set('width',
  w.add(CSS.px(20))); // 120px
    
        
    
      
    

      
    Why Typed OM?
    
      
    
        
    
          
    No more parseFloat()
    
          
    Values are already numbers with units. No string parsing needed.
    
        
    
        
    
          
    Type Safety
    
          
    Can't accidentally set width to "banana". The API validates types.
    
        
    
        
    
          
    Unit Arithmetic
    
          
    Add, subtract, multiply CSS values with proper unit handling.
    
        
    
        
    
          
    Performance
    
          
    Avoids string serialization/deserialization on every read/write.
