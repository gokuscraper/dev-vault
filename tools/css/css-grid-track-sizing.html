Grid Track Sizing Algorithm

When the browser encounters grid-template-columns, it runs a multi-phase algorithm to resolve each track's final size. The algorithm handles fixed sizes first, then intrinsic sizes, and finally flexible (fr) sizes.

Algorithm Phases

  • Phase 1: Initialize — Set fixed tracks (px, rem, %) to their exact sizes. Set base size of all other tracks to 0.
  • Phase 2: Resolve Intrinsic Sizes — For auto, min-content, max-content, fit-content(): calculate based on content.
  • Phase 3: Maximize Tracks — If there's remaining space and non-fr tracks can grow, distribute space.
  • Phase 4: Resolve Flexible (fr) Tracks — Calculate the fr unit value: remaining space ÷ total fr count. Apply to each fr track.
  • Phase 5: Apply Constraints — Enforce minmax() constraints. If a track is below its minimum, clamp it and redistribute.

Track Size Types

  • 200px — Fixed: exactly 200px regardless of container
  • 1fr — Flexible: 1 fraction of remaining space after fixed sizes are placed
  • auto — Intrinsic: fits content, but stretches to fill remaining space (like 1fr with min-content minimum)
  • minmax(100px, 1fr) — Constrained: at least 100px, at most 1fr of remaining space
  • min-content — Smallest size that doesn't cause overflow (longest word)
  • max-content — Size that fits content without wrapping
  • fit-content(200px) — Like min(max-content, max(min-content, 200px))

The fr Unit In Detail

The fr unit represents a fraction of the leftover space after all fixed, intrinsic, and percentage tracks are resolved. If the container is 600px, a fixed track takes 200px, and you have 200px 1fr 2fr:

  • Remaining space = 600 - 200 = 400px
  • Total fr = 1 + 2 = 3
  • 1fr = 400 / 3 ≈ 133.3px
  • 2fr = 266.7px

minmax() Gotchas

  • minmax(200px, 1fr) — The track is at least 200px. If 1fr resolves to less than 200px, the track stays at 200px and steals space from other fr tracks.
  • minmax(0, 1fr) — Equivalent to just 1fr (can shrink to 0)
  • minmax(auto, 300px) — At least as wide as content, at most 300px
  • If min > max, min wins (max is treated as min)
