CSS Individual Transform Properties

translate, rotate, and scale as independent CSS properties

Individual Transform Properties

CSS now supports translate, rotate, and scale as standalone properties, separate from the transform shorthand.

/* Individual properties (new) */
.element {
  translate: 50px 20px;
  rotate: 45deg;
  scale: 1.5;
}

/* Equivalent with transform shorthand (old) */
.element {
  transform: translate(50px, 20px) rotate(45deg) scale(1.5);
}

Browser support: Supported in all modern browsers since Chrome 104, Firefox 72, Safari 14.1, Edge 104. Safe for production.

transform Shorthand vs Individual Properties

The Override Problem

transform (problematic) Overrides

/* Base style */
.card {
  transform: scale(1);
}

/* Hover — OOPS, lost scale! */
.card:hover {
  transform: translateY(-5px);
  /* scale(1) is gone! */
}

Individual (clean) Independent

/* Base style */
.card {
  scale: 1;
}

/* Hover — scale preserved! */
.card:hover {
  translate: 0 -5px;
  /* scale: 1 still applies */
}

Order of Operations

Individual transform properties are always applied in a fixed order, regardless of the order they appear in CSS:

1. translate
→
2. rotate
→
3. scale
/* Even though scale is written first, translate still applies first */
.element {
  scale: 2;
  translate: 100px 0;
  rotate: 45deg;
}
/* Effective: translate(100px, 0) rotate(45deg) scale(2) */

/* With transform shorthand, order matters! */
.a { transform: rotate(45deg) translate(100px, 0); }
.b { transform: translate(100px, 0) rotate(45deg); }
/* .a and .b produce DIFFERENT results! */

3D Transforms with Individual Properties

Individual transform properties fully support 3D values.

/* 3D translate */
.element {
  translate: 50px 20px 100px;  /* x y z */
}

/* 3D rotate — axis + angle */
.element {
  rotate: x 45deg;          /* rotate around X axis */
  rotate: y 45deg;          /* rotate around Y axis */
  rotate: z 45deg;          /* same as rotate: 45deg */
  rotate: 1 1 0 45deg;      /* custom axis vector */
}

/* 3D scale */
.element {
  scale: 1.5 1 0.5;         /* x y z */
}

Independent Transitions

The biggest advantage: each individual property can have its own transition timing, duration, and easing.

/* Different timing for each transform */
.card {
  translate: 0 0;
  rotate: 0deg;
  scale: 1;

  transition:
    translate 0.3s ease-out,
    rotate 0.5s ease-in-out,
    scale 0.2s cubic-bezier(0.34, 1.56, 0.64, 1);
}

.card:hover {
  translate: 0 -10px;
  rotate: 5deg;
  scale: 1.05;
}

Animation Composition

Individual properties shine in animations — you can animate each transform independently, even from different @keyframes rules.

/* Animate only rotation — translate and scale unaffected */
@keyframes spin {
  to { rotate: 360deg; }
}

@keyframes pulse {
  0%, 100% { scale: 1; }
  50% { scale: 1.3; }
}

@keyframes bounce {
  0%, 100% { translate: 0 0; }
  50% { translate: 0 -20px; }
}

/* These can be combined! */
.icon {
  animation:
    spin 2s linear infinite,
    pulse 1.5s ease-in-out infinite,
    bounce 1s ease-in-out infinite;
}

Performance Benefits

Individual transform properties have the same compositor-level performance as the transform shorthand, with added benefits for certain use cases.

Same Performance as transform

  • Both use the GPU compositor for animations (no layout/paint triggers)
  • Both create a new stacking context and composite layer
  • will-change: translate works just like will-change: transform

Where Individual Props Win

ScenariotransformIndividual Props
Animating only rotation Must specify full transform Just rotate — simpler for compositor
Multiple animations on one element Cannot compose (one transform) Each prop gets its own animation
Cascade / component styles Overrides lose other transforms No accidental override
will-change specificity will-change: transform will-change: translate (more targeted)
Best practice: Use individual properties by default. They compose better, override safely, and enable per-property transitions. Use transform only when you need non-standard order or functions like skew(), matrix(), or perspective().

transform-only Functions

These still require the transform property:

/* No individual property equivalents */
transform: skew(15deg);
transform: matrix(1, 0, 0.5, 1, 0, 0);
transform: perspective(500px);  /* but can use perspective property on parent */

/* Individual props work WITH transform */
.element {
  translate: 50px 0;
  rotate: 10deg;
  transform: skew(5deg);  /* applied AFTER individual props */
}
