📐 CSS Layout API Playground

Experimental API Dev Toolkit Spec
The CSS Layout API is highly experimental and has no stable browser implementation. This playground uses JavaScript simulations to demonstrate how custom layout algorithms would work if the API were available.

📚 What is the CSS Layout API?

The CSS Layout API (part of CSS Houdini) allows developers to create custom layout algorithms that can be used in CSS via display: layout(name). Think of it as creating your own display: flex or display: grid from scratch.

📝 registerLayout() Structure

// layout-worklet.js
registerLayout('my-layout', class {
  // Declare which CSS properties affect this layout
  static get inputProperties() {
    return ['--columns', '--gap'];
  }

  // Declare which child CSS properties are needed
  static get childInputProperties() {
    return ['--order', '--span'];
  }

  // Intrinsic sizes calculation
  async intrinsicSizes(children, edges, styleMap) {
    // Return the minimum and preferred sizes
    const childSizes = await Promise.all(
      children.map(c => c.intrinsicSizes())
    );

    const minSize = Math.max(
      ...childSizes.map(s => s.minContentSize)
    );
    const prefSize = childSizes.reduce(
      (sum, s) => sum + s.maxContentSize, 0
    );

    return { minContentSize: minSize,
             maxContentSize: prefSize };
  }

  // The main layout algorithm
  async layout(children, edges, constraints,
               styleMap) {
    // edges: padding + border of the container
    // constraints: available width/height
    // styleMap: computed styles

    const availWidth = constraints.fixedInlineSize;
    const columns = parseInt(
      styleMap.get('--columns')) || 3;
    const gap = parseInt(
      styleMap.get('--gap')) || 10;

    const colWidth = (availWidth
      - edges.inlineStart - edges.inlineEnd
      - gap * (columns - 1)) / columns;

    const childFragments = [];
    const colHeights = new Array(columns).fill(0);

    for (const child of children) {
      // Find shortest column
      const col = colHeights.indexOf(
        Math.min(...colHeights)
      );

      // Layout the child with fixed width
      const fragment = await child.layoutNextFragment({
        fixedInlineSize: colWidth
      });

      // Position it
      fragment.inlineOffset = edges.inlineStart
        + col * (colWidth + gap);
      fragment.blockOffset = edges.blockStart
        + colHeights[col];

      colHeights[col] += fragment.blockSize + gap;
      childFragments.push(fragment);
    }

    return {
      childFragments,
      autoBlockSize: Math.max(...colHeights)
        + edges.blockEnd
    };
  }
});

🔧 CSS Usage

/* Register the worklet */
CSS.layoutWorklet.addModule('layout-worklet.js');

/* Use in CSS */
.container {
  display: layout(my-layout);
  --columns: 3;
  --gap: 16px;
}

.container > .item {
  --order: 1;
  --span: 2;
}

🏗️ Key Concepts

LayoutChild

Represents each child element. You call child.layoutNextFragment() to get its dimensions and can position it manually.

LayoutConstraints

Contains fixedInlineSize (available width) and optionally fixedBlockSize (available height).

LayoutEdges

The container's padding + border. Access via edges.inlineStart, edges.blockStart, etc.

LayoutFragment

Result of laying out a child. Has blockSize, inlineSize, inlineOffset, blockOffset.

📏 LayoutChild Intrinsic Sizes

      
The Layout API provides intrinsicSizes() to query the natural dimensions of children before positioning them. This is how the browser calculates min-content and max-content sizes.

    


    

      
Intrinsic Size Concepts

      

        
          
PropertyDescriptionExample

        
        
          

            minContentSize
            Smallest size without overflow. For text, this is the width of the longest word.
            A word "internationalization" = ~150px min-content
          

          

            maxContentSize
            Ideal size with no wrapping. For text, this is the entire line laid out.
            "Hello World" = ~85px max-content

📝 Code

      
async intrinsicSizes(children, edges, styleMap) {
  const childSizes = await Promise.all(
    children.map(child => child.intrinsicSizes())
  );

  // Min content: widest single child
  const minContentSize = Math.max(
    ...childSizes.map(s => s.minContentSize)
  ) + edges.inlineStart + edges.inlineEnd;

  // Max content: sum of all children
  const gap = parseInt(styleMap.get('--gap')) || 0;
  const maxContentSize = childSizes.reduce(
    (sum, s) => sum + s.maxContentSize, 0
  ) + gap * (children.length - 1)
    + edges.inlineStart + edges.inlineEnd;

  return { minContentSize, maxContentSize };
}

    

  


  
  

    

      
⚖️ Layout Mode Comparison

      
Compare existing CSS layout modes with what the Layout API enables.

    


    

      

        
Flexbox

        
One-dimensional layout. Items flow in a row or column with alignment and distribution controls.

      

      

        
CSS Grid

        
Two-dimensional layout. Items placed in rows and columns with explicit track sizing.

      

      

        
Layout API (Custom)

        
Arbitrary layout algorithms. Full control over positioning, sizing, and intrinsic sizes.

🛣️ Current Status

      
The CSS Layout API specification is at an early stage. Key facts:

      
    
        
  • Chrome has experimental support behind the --enable-experimental-web-platform-features flag
    • 
        
  • No Firefox or Safari implementation planned
    • 
        
  • The spec is still evolving and may change significantly
    • 
        
  • For masonry layout, CSS Grid Level 3 (native masonry) is the more likely path
    • 
        
  • Performance implications are still being studied
    • 
      

    


    

      
🔄 Alternatives for Today

      

        

          
Masonry Layout

          
Use JavaScript libraries like Masonry.js, CSS columns, or CSS Grid with grid-auto-rows + grid-row: span N.

          
.masonry {
  display: grid;
  grid-template-columns:
    repeat(auto-fill, minmax(200px, 1fr));
  grid-auto-rows: 10px;
}
.masonry-item {
  grid-row: span var(--rows);
}

        

        

          
Circular Layout

          
Use CSS transforms with rotate() and translate(), or absolute positioning with JS-calculated coordinates.

          
.circle-container {
  position: relative;
  width: 300px;
  height: 300px;
}
.circle-item {
  position: absolute;
  left: 50%;
  top: 50%;
  transform:
    rotate(calc(var(--i) * 45deg))
    translate(120px)
    rotate(calc(var(--i) * -45deg));
}
