What is a Block Formatting Context?

A BFC is a region of the page where block-level boxes are laid out. It's an isolated layout environment — elements inside a BFC don't affect layout outside, and vice versa. Think of it as a "layout boundary."

How to Create a BFC

  • display: flow-root — The cleanest way (designed for this purpose)
  • overflow: hidden | auto | scroll — Classic hack (may clip content)
  • float: left | right — Creates BFC but also floats the element
  • position: absolute | fixed — Creates BFC but removes from flow
  • display: inline-block — Creates BFC but changes display behavior
  • display: flex | grid — Children are in flex/grid formatting context, but the container creates a BFC
  • contain: layout | content | paint — CSS Containment creates a BFC
  • column-count or column-width (not auto)

What BFC Does

  • Contains floats — A BFC parent wraps around its floated children (no height collapse)
  • Prevents margin collapse — Margins of elements in different BFCs don't collapse
  • Excludes external floats — Content in a BFC doesn't wrap around external floats
  • Layout isolation — Internal layout doesn't affect siblings or parents

Common Pain Points Solved by BFC

  • "Why is my container 0px tall?" → Floated children escape the parent. Create a BFC on the parent.
  • "Why are my margins collapsing?" → Adjacent margins collapse in the same BFC. Create a new BFC to prevent it.
  • "Why is my text wrapping around the float?" → The text's container is in the same BFC as the float. Create a BFC on the text container to make it sit beside the float.
