01 Basic Function

@function defines a reusable CSS function. It accepts parameters via custom properties, performs calculations, and returns a value with result:.

Definition & Usage
/* Define a function that doubles a value */
@function --double(--value) {
  result: calc(var(--value) * 2);
}

/* Use it in any property */
.box {
  --base: 10px;
  padding: --double(var(--base));       /* 20px */
  margin:  --double(16px);              /* 32px */
  border-radius: --double(4px);       /* 8px */
}

Input

How it works: Functions are called with the --name(args) dashed-function syntax. The result: descriptor sets the return value. Unlike JavaScript, there is no early return — the last result: in cascade order wins.
Default Parameters
@function --add-margin(--size: 10px) {
  result: calc(var(--size) + 10px);
}

.a { margin: --add-margin();           /* 20px (default) */ }
.b { margin: --add-margin(24px);       /* 34px */ }

02 Type System

Parameters and return values can be typed. Invalid types are rejected at computed-value time. Use type() for union types.

Typed Parameters & Return
@function --scale(--base <length>, --factor <number>: 2) returns <length> {
  result: calc(var(--base) * var(--factor));
}

.box {
  width: --scale(100px, 1.5);      /* 150px - valid */
  height: --scale(red, 2);         /* invalid - <color> is not <length> */
}

/* Union types with type() */
@function --flex-size(--val type(<length> | <percentage>)) returns type(<length> | <percentage>) {
  result: var(--val);
}

Distances: px, em, rem, vw, vh...
--pad <length>: 16px
<color>

<percentage>
Percentage value

<angle>

--rot <angle>: 45deg
<integer>

<time>

03 Conditional Logic

Functions use cascade resolution. Nest @media to branch on viewport. The upcoming if() function enables inline conditionals.

@media Inside Functions
/* Return different values based on viewport width */
@function --responsive-gap(--narrow <length>, --wide <length>) returns <length> {
  result: var(--wide);

  @media (width < 700px) {
    result: var(--narrow);
  }
}

.grid {
  gap: --responsive-gap(0.5rem, 2rem);
}
Cascade, not early return: Unlike if/else in JavaScript, CSS functions evaluate all result: declarations. The one that wins in cascade order (including @media applicability) becomes the return value.
Layout Sidebar (Responsive Pattern)
@function --layout-sidebar(--sidebar-width: 20ch) {
  result: 1fr;

  @media (width > 640px) {
    result: var(--sidebar-width) auto;
  }
}

.layout {
  display: grid;
  grid-template-columns: --layout-sidebar();
}

Narrow (<640px)

grid-template-columns: 1fr

Wide (>640px)

grid-template-columns: 20ch auto
if() Function (Inline Conditional)
/* if() provides inline branching (part of CSS Values L5) */
@function --theme-color(--variant) {
  result: if(media(prefers-color-scheme: dark): #e2e8f0 ; else: #1a1a2e);
}

/* Style queries inside if() */
@function --adaptive-size(--type: 'default') {
  result: if(
    style(--type: 'compact'): 0.875rem ;
    style(--type: 'large'):   1.25rem ;
    else:                     1rem
  );
}
Note: The if() function is a separate CSS proposal (CSS Values Level 5) that complements @function. Browser support may differ.

04 Color Functions

Build custom color manipulation utilities — opacity control, tinting, shade generation — using relative color syntax inside @function.

Opacity Function
@function --opacity(--color <color>, --alpha <percentage>: 50%) returns <color> {
  result: rgb(from var(--color) r g b / var(--alpha));
}

05 Spacing Scale

Create a consistent spacing system with a single function. Multiply a base unit by a named step to produce harmonious spacing values.

Multiplier-Based Scale
@function --space(--step <number>: 1) returns <length> {
  --base: 0.25rem;  /* 4px */
  result: calc(var(--base) * var(--step));
}

.card {
  padding:    --space(4);    /*  1rem = 16px */
  gap:        --space(3);    /* .75rem = 12px */
  margin-bottom: --space(6); /* 1.5rem = 24px */
}

.section {
  padding: --space(8) --space(4);  /* 2rem 1rem */
}

06 Fluid Typography

Build a fluid font-size function using clamp() that smoothly scales between a minimum and maximum size.

Fluid Size Function
@function --fluid(
  --min <length>,
  --max <length>,
  --min-vw <length>: 20rem,
  --max-vw <length>: 80rem
) returns <length> {
  --slope: calc(
    (var(--max) - var(--min)) /
    (var(--max-vw) - var(--min-vw))
  );
  --intercept: calc(
    var(--min) - var(--slope) * var(--min-vw)
  );
  result: clamp(
    var(--min),
    var(--intercept) + var(--slope) * 100vw,
    var(--max)
  );
}

h1 { font-size: --fluid(1.5rem, 3rem); }
h2 { font-size: --fluid(1.25rem, 2rem); }
p  { font-size: --fluid(0.9rem, 1.1rem); }
--fluid(1.5rem, 3rem)
The quick brown fox jumps
--fluid(1.25rem, 2rem)
Over the lazy dog near the river
--fluid(0.9rem, 1.1rem)
Body text that smoothly adapts between viewport widths. Resize your browser to see the effect. The function calculates a linear slope between the min and max font sizes.
Why this matters: Instead of writing clamp() math for every element, you call --fluid(min, max). Change the base formula once, and every usage updates. This is the kind of DRY pattern that was previously only possible with preprocessors.

Same Pattern, Three Ways

CSS @function

@function --size(--step <number>) returns <length> {
  result: calc(0.25rem * var(--step));
}
.box { padding: --size(4); }

CSS Custom Properties (workaround)

:root {
  --size-1: 0.25rem;
  --size-2: 0.5rem;
  --size-3: 0.75rem;
  --size-4: 1rem;
  /* Must define every step manually */
}
.box { padding: var(--size-4); }

Sass Function

@function size($step) {
  @return 0.25rem * $step;
}
.box { padding: size(4); }
// Compiled at build time - cannot respond to runtime state

How This Playground Works Without @function

All visual demonstrations on this page use standard CSS (calc(), clamp(), color-mix()) to simulate the output of @function calls. The code examples show the actual @function syntax, while the previews show what the computed result would be. In the Live Editor (section 07), if your browser supports @function natively, the preview renders real CSS; otherwise it applies a basic JS-based fallback that processes simple function calls.
