1 Accordion

Collapsible sections that smoothly animate between height: 0 and height: auto.

interpolate-size: allow-keywords
Live Demo
interpolate-size is a CSS property that enables smooth transitions to and from intrinsic sizing keywords like auto, min-content, max-content, and fit-content. Previously, animating height: auto required JavaScript to measure the element's actual height.
Set interpolate-size: allow-keywords on a parent element (or :root). The browser will then interpolate between a fixed length (like 0) and an intrinsic keyword (like auto) during transitions and animations. The browser resolves the keyword at each frame of the animation.
Yes! Add :root { interpolate-size: allow-keywords; } to enable it everywhere. This is safe as a progressive enhancement; browsers that don't support it will simply skip the animation and snap to the final state. 
.accordion {
  interpolate-size: allow-keywords;
}
.panel {
  height: 0;
  overflow: hidden;
  transition: height 0.4s ease;
}
.panel.open {
  height: auto;  /* This now animates! */
}

3 Details / Summary

Native HTML <details> element with smooth open/close animation.

interpolate-size: allow-keywords
Live Demo
What sizing keywords can be animated?
With interpolate-size: allow-keywords, you can animate between fixed lengths and: auto, min-content, max-content, fit-content, stretch, and contain. You can also use calc-size() to perform calculations with these keywords.
Is it a progressive enhancement?
Yes. In browsers that don't support interpolate-size, the content will simply snap to its final state without animation. The functionality (show/hide) still works. This makes it safe to use in production today.
Does it work with CSS animations too?
Absolutely. interpolate-size works with both CSS transitions and CSS @keyframes animations. You can animate from height: 0 to height: auto in a keyframe sequence just as easily as with a transition. 
details {
  interpolate-size: allow-keywords;
}
.content {
  height: 0;
  overflow: hidden;
  transition: height 0.4s ease;
}
details[open] .content {
  height: auto;
}

5 calc-size()

Perform calculations with intrinsic size keywords. calc-size(auto, size / 2) gives you half the auto width.

Click buttons to switch sizes
Hello, calc-size()! 
/* Half of the element's auto width */
.box {
  width: calc-size(auto, size / 2);
}

/* Auto width plus 40px padding */
.box {
  width: calc-size(auto, size + 40px);
}

/* 75% of fit-content width */
.box {
  width: calc-size(fit-content, size * 0.75);
}

7 Before / After

The old JavaScript approach vs. the new CSS-only solution.

✗ Old: JavaScript
// Measure, set, animate, clean up
function togglePanel(el) {
  if (el.style.height) {
    el.style.height = '';
  } else {
    const h = el.scrollHeight;
    el.style.height = h + 'px';
    el.addEventListener('transitionend',
      () => el.style.height = 'auto',
      { once: true }
    );
  }
}

// Also need: ResizeObserver for
// dynamic content, cleanup logic,
// edge-case handling...
✓ New: CSS Only
:root {
  interpolate-size: allow-keywords;
}

.panel {
  height: 0;
  overflow: hidden;
  transition: height 0.4s ease;
}

.panel.open {
  height: auto;
}

/* That's it. No JS needed.
   Works with any intrinsic keyword.
   Handles dynamic content.
   Progressive enhancement built-in. */
