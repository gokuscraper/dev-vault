Environment Variables

What is a .env file?

A .env file stores environment variables — configuration values that change between environments (development, staging, production). It keeps secrets out of source code.

Best Practices

1. Never commit .env to git. Add it to .gitignore.

2. Create a .env.example with placeholder values (no real secrets) and commit that.

3. Use different .env files per environment: .env.development, .env.production.

4. Use strong, random values for secrets. Never reuse passwords.

5. Keep variable names UPPER_SNAKE_CASE by convention.

6. Quote values that contain spaces: APP_NAME="My App".

Loading .env files

Node.js: npm install dotenv → require('dotenv').config()

Python: pip install python-dotenv → from dotenv import load_dotenv; load_dotenv()

Next.js / Vite: Loaded automatically. Prefix with NEXT_PUBLIC_ or VITE_ for client-side access.

Docker: docker run --env-file .env myapp

Ruby: gem 'dotenv-rails' → loaded automatically in development.
